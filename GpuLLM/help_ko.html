GpuLLM으로 할 수 있는 것

GpuLLM은 PC를 완전한 로컬 AI 워크스테이션으로 바꾸는 Windows 데스크톱 애플리케이션입니다. 모든 계산은 사용자의 하드웨어에서 실행되며 — 데이터가 기기를 떠나는 일은 없습니다.

🧠

LLM을 로컬에서 실행

GGUF 형식 모델을 로드하고 GPU에서 직접 대화하세요. CUDA, Vulkan, CPU 백엔드를 지원하며 자동 폴백 기능 포함.

📊

하드웨어 모니터링

실시간 GPU 사용률, VRAM, 온도, 소비 전력, CPU 사용률, 메모리 — 60초 스파크라인 차트가 포함된 하나의 대시보드에서 모두 확인.

🔬

모델 벤치마크

추론 속도(TTFT, tokens/sec, 레이턴시)를 측정하고 내장된 MMLU 및 C-Eval 문제 세트로 모델 정확도를 평가.

📦

모델 다운로드 및 관리

HuggingFace의 13개 큐레이션 모델 카탈로그를 탐색하고, 재개 지원 및 SHA-256 검증으로 다운로드하거나, 자신의 GGUF 파일을 가져오세요.

💬

듀얼 모델 채팅

하나의 대화에서 두 모델을 나란히 비교하세요. LLM1이 답변하고 LLM2가 검토 — 두 모델 모두 전체 컨텍스트를 확인.

😺

프리즘 캣 마스코트

GPU 워크로드에 따라 색상과 감정이 변화하는 라이브 애니메이션 GPU 스피릿.

100% 로컬 & 비공개: 모델이 다운로드되면 추론에 인터넷 연결이 필요하지 않습니다. 텔레메트리 없음, 클라우드 의존성 없음, 데이터 업로드 없음.

시스템 요구 사항

설치하기 전에 PC가 다음 요구 사항을 충족하는지 확인하세요.

운영 체제Windows 10 (21H2 이상) / Windows 11

고성능 GPU가 없으신가요? GpuLLM은 CPU 전용 시스템에서도 작동합니다 — 추론 속도는 더 느립니다. Llama 3.2 1B 또는 DeepSeek-R1-Distill 1.5B와 같은 1-3B 파라미터 모델부터 시작하세요.

빠른 시작 — 최초 설정

다음 단계에 따라 처음부터 첫 로컬 LLM 추론을 실행하세요.

  1. GpuLLM 다운로드 및 설치

    Microsoft Store에서 GpuLLM을 다운로드하세요. 앱은 자체 완결형으로 추가 런타임이나 CUDA 툴킷이 필요하지 않습니다.

  2. HuggingFace 액세스 토큰 구성 (선택 사항)

    설정을 열고 HuggingFace Read 토큰을 입력하세요. 이는 게이트 모델(Qwen, Meta-Llama 등)에만 필요합니다. bartowski/*의 커뮤니티 모델은 토큰 없이 작동합니다.

    토큰 생성 방법: huggingface.co/settings/tokens로 이동하여 New token을 클릭하고, 유형 Read를 선택한 후 이름(예: "GpuLLM")을 지정하고 생성된 토큰을 복사하세요. GpuLLM 설정에 붙여넣으세요. 토큰은 Windows DPAPI로 암호화되어 디스크에 기록되지 않습니다.
  3. 모델 다운로드

    모델 라이브러리로 이동하여 큐레이션 카탈로그를 탐색하고, 모델 카드에서 다운로드를 클릭하세요. 내 하드웨어 버튼으로 GPU의 VRAM과 호환되는 모델을 필터링하세요.

  4. 모델 로드

    다운로드 후, 모델 카드에서 로드를 클릭하세요. 모델 가중치가 GPU 메모리에 로드되어 빠른 추론을 위해 상주합니다.

  5. 채팅 시작

    채팅 벤치마크로 전환하고 프롬프트를 입력한 후 Enter 키를 누르세요. 모델이 실시간으로 응답을 스트리밍하며 사이드바에서 GPU 메트릭이 실시간 업데이트됩니다.

프로 팁: 더 큰 모델로 이동하기 전에 작은 모델(Llama 3.2 1B 또는 DeepSeek-R1-Distill 1.5B 등)로 모든 것이 정상 작동하는지 확인하세요.

일반적인 사용 시나리오

어디서부터 시작할지 모르겠나요? 다음은 참고할 만한 일반적인 워크플로우입니다.

🎯 "내 GPU에 가장 적합한 가장 빠른 모델을 찾고 싶습니다"

모델 라이브러리로 이동하여 내 하드웨어를 클릭해 호환 모델을 필터링하고, 몇 개를 다운로드한 후 각각 로드하여 채팅 벤치마크에서 tokens/sec 점수를 비교하세요. 각 추론 후 벤치마크 KPI 카드를 확인하세요.

🤝 "두 모델을 나란히 비교하고 싶습니다"

두 모델을 로드하고 채팅 벤치마크로 전환한 후 듀얼 모드를 토글하고 하나는 응답자, 다른 하나는 리뷰어로 할당하세요. 두 모델 모두 동일한 프롬프트에 응답 — 실시간으로 답변을 비교하세요.

📊 "내 모델의 정확도를 알고 싶습니다"

모델을 로드하고 평가로 이동하여 MMLU 또는 C-Eval을 선택한 후 시작을 클릭하세요. 카테고리별 정확도 점수와 난이도 수준 분석을 확인할 수 있습니다 — Markdown으로 내보내기 가능.

💰 "클라우드 API 대비 얼마나 절약되는지 보고 싶습니다"

채팅 벤치마크에서 추론을 실행하세요. 내장된 비용 추정기가 로컬 추론 비용(전기료 + 하드웨어 감가상각)을 OpenAI 및 Anthropic API 가격과 자동 비교합니다. 세션 보고서에서 누적 절감액을 확인하세요.

📥 "이미 GGUF 파일이 있습니다 — 사용할 수 있나요?"

네. 모델 라이브러리에서 모델 가져오기를 클릭하고 GGUF 파일을 선택하거나, 창에 끌어다 놓으세요. GpuLLM이 재다운로드 없이 라이브러리에 추가합니다.

앱 네비게이션

GpuLLM에는 왼쪽 사이드바에서 접근할 수 있는 5개의 메인 페이지가 있습니다. 아이콘을 클릭하여 페이지를 전환하세요.

채팅 벤치마크

메인 추론 및 벤치마크 페이지입니다. 여기서 할 수 있는 것:

  • 모델과 채팅: 작업 입력 상자에 입력하고 Enter (또는 Ctrl+Enter)를 눌러 전송. 모델이 토큰별로 응답을 스트리밍합니다.
  • GPU 메트릭 모니터링: 오른쪽 패널에 실시간 GPU 사용률, VRAM 사용량, 온도, 소비 전력, CPU 사용률, 시스템 메모리가 표시 — 모두 60초 스파크라인 트렌드 포함.
  • 추론 파라미터 조정: 상단의 슬라이더로 각 요청의 최대 토큰과 온도를 설정.
  • 벤치마크 결과 확인: 각 추론 후 KPI 카드에 TTFT(첫 토큰 대기 시간), 초당 토큰, 레이턴시, 백엔드 체인, GPU 피크 메트릭이 표시.
  • 채팅 내보내기: PDF 내보내기를 클릭하여 대화와 벤치마크 보고서를 A4 PDF로 저장.
  • 메시지 복사: 각 메시지 버블에 복사 버튼과 타임스탬프(HH:mm:ss)가 있습니다.

듀얼 모델 대화

채팅 벤치마크에서 듀얼 모드를 토글하여 두 모델 릴레이 추론을 활성화:

  • 드롭다운에서 주 모델(LLM1)과 보조 모델(LLM2)을 선택.
  • LLM1이 프롬프트를 받아 답변을 생성.
  • LLM2가 자동으로 전체 대화 컨텍스트를 받아 검토 또는 세컨드 오피니언 제공.
  • 메시지는 역할별로 레이블: LLM1 · 응답자 및 LLM2 · 리뷰어.

평가

표준화된 문제 세트를 사용하여 로드된 모델에 대해 구조화된 벤치마크 실행:

  • MMLU: 57개 과목(STEM, 인문, 사회과학)에 걸친 100개 샘플 문제.
  • C-Eval: 중국어 학술 평가 벤치마크의 100개 샘플 문제.
  • 페이지는 세 단계: 구성 (모델 + 벤치마크 선택) → 실행 중 (실시간 진행률) → 결과 (카테고리 및 난이도별 정확도).
  • 채점 시작 전에 웜업 추론이 실행되어 모델이 완전히 로드되었는지 확인.

모델 라이브러리

GGUF 모델 컬렉션 탐색, 다운로드 및 관리:

  • 필터 필: 카테고리별 필터링 — 코딩, 채팅, 추론.
  • 내 하드웨어: GPU가 실행할 수 있는 모델만 표시하는 원클릭 필터. 호환 모델에 녹색 체크 표시.
  • 다운로드: 카드의 다운로드 버튼 클릭. GpuLLM이 라이선스 동의, 파일 목록, 재개 가능 다운로드, SHA-256 검증을 자동 처리.
  • 가져오기: 드래그 앤 드롭 또는 "모델 가져오기"를 클릭하여 기존 GGUF 파일 추가.
  • 관리: 폴더 열기, 경로 복사, 카드에서 모델 로드/언로드.

설정

기본 설정에 맞게 애플리케이션 구성:

  • 테마: 선호하는 비주얼 테마 선택.
  • 다운로드 디렉토리: 모델 파일이 저장될 위치 선택.
  • HuggingFace 토큰: HuggingFace 액세스 토큰 입력 또는 삭제 (Windows Credential Manager에 안전하게 저장).
  • 법적 링크: 개인정보 처리방침 및 서드파티 공지 접근.

모델 다운로드 이해하기

게이트 모델 vs. 오픈 모델

모델 카드에는 인증 필요 여부를 나타내는 배지가 표시됩니다:

  • 🔒 게이트 — HuggingFace 액세스 토큰 필요. Qwen, meta-llama 등의 공식 리포지토리에서 일반적.
  • 오픈 (배지 없음) — 인증 불필요. bartowski/*, lmstudio-community/*, TheBloke/*의 커뮤니티 재패키지.

다운로드 프로세스

다운로드를 클릭하면 GpuLLM이 자동으로 다음을 수행:

  1. 라이선스 동의 확인 (허용적 라이선스는 자동 통과, 제한적 라이선스는 대화상자 표시).
  2. HuggingFace API에 쿼리하여 올바른 GGUF 파일 식별.
  3. HTTP 재개 지원 다운로드 (.part 임시 파일).
  4. HuggingFace LFS 메타데이터 대비 SHA-256 해시 검증.
토큰이 필요하신가요? "액세스 토큰 필요"로 다운로드가 실패하면 huggingface.co/settings/tokens에서 HuggingFace Read 토큰을 생성하여 설정에 입력하세요. 토큰은 Windows Credential Manager(DPAPI 암호화)에 저장되며 디스크에 기록되지 않습니다.

다운로드 관리

  • 취소: 다운로드 중인 카드에서 취소 클릭. 부분 .part 파일은 이후 재개를 위해 보존.
  • 위치 변경: 설정 → 다운로드 디렉토리에서 새 폴더 선택.
  • 폴더 열기: 다운로드된 카드에서 폴더 아이콘 클릭하여 탐색기에서 파일 위치 열기.
  • GGUF 가져오기: "모델 가져오기" 버튼 또는 드래그 앤 드롭으로 기존 GGUF 파일 추가.

추론 및 벤치마크 상세

백엔드 선택

GpuLLM은 자동으로 하드웨어를 감지하고 다음 순서로 가장 빠른 사용 가능한 백엔드를 선택: CUDA 12 → Vulkan → CPU. 수동 구성 불필요 — 각 추론 후 벤치마크 결과에 백엔드 체인이 표시.

모델 로드 전략

모델 카드에서 로드를 클릭하면 모델 가중치가 GPU/CPU 메모리에 사전 로드되어 상주. 이후 모든 추론 호출은 사전 로드된 가중치를 재사용하여 디스크 I/O를 완전히 건너뛰고 빠른 응답 시간 달성. 한 번에 하나의 모델만 로드 가능(듀얼 모드에서는 두 개).

벤치마크 메트릭

각 추론 후 GpuLLM은 다음을 보고:

  • TTFT — 첫 토큰 대기 시간: 첫 응답 토큰이 나타날 때까지의 대기 시간.
  • Tokens/s — 첫 토큰 이후 생성 처리량.
  • 레이턴시 — 종단 간 총 응답 시간.
  • GPU 피크 — 추론 중 최대 GPU 사용률, VRAM 사용량, 온도.
  • 백엔드 체인 — 사용된 백엔드 (예: "CUDA 12 → Vulkan → CPU 폴백").

세션 보고서는 앱 시작 이후 모든 추론 메트릭을 집계 (누적 토큰, 평균 레이턴시, 피크 GPU 값).

팁 및 문제 해결

키보드 단축키

  • Enter 또는 Ctrl+Enter — 채팅 벤치마크에서 프롬프트 전송.

모범 사례

  • 작게 시작: 더 많은 VRAM이 필요한 큰 모델을 로드하기 전에 1-3B 파라미터 모델로 설정 확인.
  • VRAM 확인: 모델 라이브러리의 내 하드웨어 필터를 사용하여 GPU VRAM에 맞는 모델만 보기. 녹색 체크 표시가 호환성 확인.
  • 온도 조정: 사실/코딩 작업에는 낮은 온도(0.1–0.5); 창의적 글쓰기에는 높은 온도(0.7–1.0). 채팅 벤치마크 슬라이더에서 작업별로 조정.
  • 최대 토큰: 합리적으로 설정 — 너무 낮으면 응답이 잘리고 너무 높으면 시간 낭비. 대부분의 작업에 512–2048 토큰이면 충분.
  • 듀얼 모델 비교: 듀얼 모드를 두 개의 다른 모델(예: 코딩 모델 + 범용 모델)과 함께 사용하여 동일한 프롬프트에서 답변 비교.
  • 평가 정확도: 더 결정적이고 공정한 채점을 위해 낮은 온도(0.1–0.3)로 평가 실행.
  • 토큰 보안: HF 토큰은 Windows DPAPI를 통해 Credential Manager에 암호화 저장 — 파일에 기록되지 않음. huggingface.co/settings/tokens에서 언제든지 철회 가능.

문제 해결

  • 모델 다운로드 실패 401/403: 모델이 게이트되어 있음. huggingface.co/settings/tokens에서 HuggingFace Read 토큰 생성 후 설정에 입력하고 재시도.
  • 다운로드 시작 후 "무결성 검증 실패"로 실패: 전송 중 파일 손상. 다시 다운로드 클릭 — GpuLLM이 새 다운로드 시작.
  • 속도 제한 / HTTP 429: HuggingFace가 익명 API 액세스 제한. 더 높은 속도 제한을 위해 토큰 구성.
  • 재시작 후 모델이 "다운로드됨"으로 표시되지 않음: 다운로드 디렉토리가 변경되었을 수 있음. 설정 → 다운로드 디렉토리 확인 또는 가져오기 버튼 사용.
  • 추론 속도 느림: CUDA 백엔드가 활성 상태인지 확인 (벤치마크 결과에 표시됨). CPU로 폴백 중이면 NVIDIA 드라이버가 최신인지 확인.
  • CUDA 미감지 / 백엔드가 CPU로 폴백: NVIDIA 드라이버를 545 이상으로 설치 또는 업데이트. 노트북 GPU의 경우 GpuLLM이 전용 GPU에서 실행 중인지 확인.
  • "메모리 부족" 또는 모델 로드 실패: GPU VRAM 부족. 더 작은 모델 또는 더 높은 압축 양자화 시도. 내 하드웨어로 호환 모델 필터링. 다른 GPU 사용 앱 종료.
  • 모델 로드 후 의미 없는 텍스트 생성: GGUF 파일 손상 또는 형식 오류 가능성. 다운로드 후 SHA-256 해시 검증. 모델이 GGUF 형식인지 확인.
  • 듀얼 모드에서 동일 모델이 두 번 표시: 듀얼 모드 활성화 전에 두 개의 다른 모델 로드 필요. 먼저 LLM1 로드 후 모델 라이브러리에서 LLM2 로드.

용어집

GGUFllama.cpp 프로젝트가 설계한 양자화된 대규모 언어 모델 파일 형식. GpuLLM은 GGUF 형식만 사용합니다.
양자화수치 정밀도(예: 16비트에서 4비트로)를 낮춰 모델 메모리 사용량을 줄이는 기술. 낮은 양자화는 더 적은 VRAM을 사용하지만 품질이 약간 저하될 수 있습니다. 일반 형식: Q8_0, Q4_K_M, Q2_K.
Tokens/s초당 생성되는 토큰 수 — GPU 성능을 측정하는 주요 지표. 값이 높을수록 빠른 추론을 의미.
TTFT첫 토큰 대기 시간 — 모델이 첫 응답 토큰을 생성하는 데 걸리는 시간. 사용자 체감 지연 시간을 측정.
MMLU대규모 멀티태스크 언어 이해 — 57개 과목에 걸친 표준화된 LLM 능력 벤치마크.
C-Eval중국어 학술 평가 벤치마크 — 중국어 LLM 능력을 위한 MMLU의 중국어 대응 벤치마크.
VRAM비디오 RAM — GPU 전용 메모리. 모델 로딩과 추론 모두 VRAM을 소비. 더 큰 모델은 더 많은 VRAM 필요.
백엔드모델 추론을 실행하는 컴퓨트 엔진. GpuLLM은 CUDA(NVIDIA GPU), Vulkan(크로스벤더 GPU), CPU(폴백) 지원.
