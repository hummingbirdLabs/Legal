Ce que GpuLLM peut faire

GpuLLM est une application de bureau Windows qui transforme votre PC en une station de travail IA locale complète. Tous les calculs s'exécutent sur votre matériel — rien ne quitte votre appareil.

🧠

Exécuter des LLM localement

Chargez des modèles au format GGUF et dialoguez directement sur votre GPU. Prend en charge les backends CUDA, Vulkan et CPU avec basculement automatique.

📊

Surveiller le matériel

Utilisation GPU en temps réel, VRAM, température, consommation électrique, utilisation CPU et mémoire — le tout dans un tableau de bord avec graphiques sparkline de 60 secondes.

🔬

Benchmarker des modèles

Mesurez la vitesse d'inférence (TTFT, tokens/s, latence) et évaluez la précision des modèles avec les ensembles de questions MMLU et C-Eval intégrés.

📦

Télécharger et gérer des modèles

Parcourez un catalogue organisé de 13 modèles HuggingFace, téléchargez avec reprise et vérification SHA-256, ou importez vos propres fichiers GGUF.

💬

Chat Double Modèle

Comparez deux modèles côte à côte dans une seule conversation. LLM1 répond, LLM2 évalue — les deux voient le contexte complet.

😺

Mascotte Prism Cat

Un esprit GPU animé en direct qui change de couleur et d'émotion en fonction de la charge de travail de votre GPU.

100 % Local & Privé : Une fois les modèles téléchargés, aucune connexion Internet n'est requise pour l'inférence. Pas de télémétrie, pas de dépendance cloud, pas de téléchargement de données.

Pas de GPU puissant ? GpuLLM fonctionne aussi sur des systèmes CPU uniquement — attendez-vous juste à une inférence plus lente. Commencez par des modèles 1-3B comme Llama 3.2 1B ou DeepSeek-R1-Distill 1.5B.

Démarrage rapide — Première configuration

Suivez ces étapes pour passer de zéro à votre première inférence LLM locale.

  1. Télécharger et installer GpuLLM

    Obtenez GpuLLM depuis le Microsoft Store. L'application est autonome — aucun runtime supplémentaire ni CUDA toolkit requis.

  2. Configurer votre token d'accès HuggingFace (optionnel)

    Ouvrez Paramètres et saisissez un token Read HuggingFace. Cela n'est nécessaire que pour les modèles protégés (ex. Qwen, Meta-Llama). Les modèles communautaires de bartowski/* fonctionnent sans token.

    Comment créer un token : Allez sur huggingface.co/settings/tokens, cliquez sur New token, sélectionnez le type Read, donnez-lui un nom (ex. "GpuLLM") et copiez le token généré. Collez-le dans les Paramètres GpuLLM. Le token est stocké chiffré dans le Gestionnaire d'informations d'identification Windows (DPAPI) et jamais journalisé ni écrit sur le disque.
  3. Télécharger un modèle

    Naviguez vers Bibliothèque de modèles, parcourez le catalogue organisé et cliquez sur Télécharger sur une fiche de modèle. Utilisez le bouton Mon Matériel pour filtrer les modèles compatibles avec la VRAM de votre GPU.

  4. Charger le modèle

    Une fois téléchargé, cliquez sur Charger sur la fiche du modèle. Les poids du modèle sont chargés dans la mémoire GPU et restent résidents pour une inférence rapide.

  5. Commencer à chatter

    Passez à Chat Benchmark, tapez votre prompt et appuyez sur Entrée. Le modèle diffuse sa réponse en temps réel tandis que les métriques GPU se mettent à jour en direct dans la barre latérale.

Conseil de pro : Commencez par un modèle plus petit (ex. Llama 3.2 1B ou DeepSeek-R1-Distill 1.5B) pour vérifier que tout fonctionne avant de passer à des modèles plus grands.

Scénarios d'utilisation courants

Vous ne savez pas par où commencer ? Voici quelques flux de travail typiques pour vous guider.

🎯 "Je veux trouver le modèle le plus rapide pour mon GPU"

Allez dans Bibliothèque de modèles, cliquez sur Mon Matériel pour filtrer les modèles compatibles, téléchargez-en quelques-uns, chargez chacun et utilisez Chat Benchmark pour comparer leurs scores tokens/s. Vérifiez les cartes KPI de benchmark après chaque inférence.

🤝 "Je veux comparer deux modèles côte à côte"

Chargez deux modèles, passez à Chat Benchmark, activez le Mode Double et attribuez l'un comme Répondeur et l'autre comme Évaluateur. Les deux modèles répondent au même prompt — comparez les réponses en temps réel.

📊 "Je veux savoir à quel point mon modèle est précis"

Chargez un modèle, naviguez vers Évaluation, choisissez MMLU ou C-Eval et cliquez sur Démarrer. Vous obtiendrez des scores de précision par catégorie et une répartition par niveau de difficulté — exportable en Markdown.

💰 "Je veux voir combien d'argent j'économise par rapport aux API cloud"

Exécutez n'importe quelle inférence dans Chat Benchmark. L'estimateur de coût intégré compare automatiquement votre coût d'inférence local (électricité + amortissement matériel) avec les tarifs API OpenAI et Anthropic. Consultez le rapport de session pour les économies cumulées.

📥 "J'ai déjà des fichiers GGUF — puis-je les utiliser ?"

Oui. Dans la Bibliothèque de modèles, cliquez sur Importer un modèle et sélectionnez votre fichier GGUF, ou faites simplement un glisser-déposer sur la fenêtre. GpuLLM l'ajoute à votre bibliothèque sans le re-télécharger.

Navigation dans l'application

GpuLLM a cinq pages principales accessibles depuis la barre latérale gauche. Cliquez sur une icône pour changer de page.

Chat Benchmark

La page principale d'inférence et de benchmarking. Ici, vous pouvez :

  • Chatter avec un modèle : Tapez dans la zone Saisie de tâche et appuyez sur Entrée (ou Ctrl+Entrée) pour envoyer. Le modèle diffuse sa réponse token par token.
  • Surveiller les métriques GPU : Le panneau de droite affiche l'utilisation GPU en temps réel, l'utilisation VRAM, la température, la consommation électrique, l'utilisation CPU et la mémoire système — le tout avec des tendances sparkline de 60 secondes.
  • Ajuster les paramètres d'inférence : Utilisez les curseurs en haut pour définir Max Tokens et Temperature pour chaque requête.
  • Voir les résultats de benchmark : Après chaque inférence, les cartes KPI affichent TTFT (Time-To-First-Token), tokens par seconde, latence, chaîne de backend et métriques GPU de pointe.
  • Exporter le chat : Cliquez sur Exporter PDF pour enregistrer la conversation et le rapport de benchmark en PDF A4.
  • Copier les messages : Chaque bulle de message a un bouton de copie et un horodatage (HH:mm:ss).

Conversation Double Modèle

Activez le Mode Double dans Chat Benchmark pour activer l'inférence relais à deux modèles :

  • Sélectionnez un modèle principal (LLM1) et un modèle secondaire (LLM2) dans les menus déroulants.
  • LLM1 reçoit votre prompt et génère une réponse.
  • LLM2 reçoit automatiquement le contexte complet de la conversation et fournit un avis ou un deuxième avis.
  • Les messages sont étiquetés par rôle : LLM1 · Répondeur et LLM2 · Évaluateur.

Évaluation

Exécutez des benchmarks structurés sur des modèles chargés à l'aide d'ensembles de questions standardisés :

  • MMLU : 100 questions échantillonnées dans 57 matières (STEM, sciences humaines, sciences sociales).
  • C-Eval : 100 questions échantillonnées à partir des benchmarks d'évaluation académique chinois.
  • La page a trois phases : Configurer (choisir modèle + benchmark) → Exécution (progression en temps réel) → Résultats (précision par catégorie et difficulté).
  • Une inférence d'échauffement est d'abord exécutée pour garantir que le modèle est complètement chargé avant le début de l'évaluation.

Bibliothèque de modèles

Parcourez, téléchargez et gérez votre collection de modèles GGUF :

  • Filtres : Affinez par catégorie — Coding, Chat ou Reasoning.
  • Mon Matériel : Filtre en un clic pour n'afficher que les modèles que votre GPU peut exécuter. Les modèles compatibles affichent une coche verte.
  • Télécharger : Cliquez sur le bouton Télécharger d'une fiche. GpuLLM gère automatiquement le consentement de licence, la liste des fichiers, le téléchargement avec reprise et la vérification SHA-256.
  • Importer : Par glisser-déposer ou en cliquant sur "Importer un modèle" pour ajouter tout fichier GGUF existant.
  • Gérer : Ouvrir le dossier, copier le chemin ou charger/décharger des modèles depuis leurs fiches.

Paramètres

Configurez l'application selon vos préférences :

  • Thème : Choisissez votre thème visuel préféré.
  • Répertoire de téléchargement : Choisissez où les fichiers modèles sont stockés.
  • Token HuggingFace : Saisissez ou effacez votre token d'accès HuggingFace (stocké en sécurité dans le Gestionnaire d'informations d'identification Windows).
  • Liens légaux : Accédez à la Politique de confidentialité et aux Avis tiers.

Comprendre les téléchargements de modèles

Modèles protégés vs ouverts

Les fiches de modèle affichent un badge indiquant si une authentification est requise :

  • 🔒 Protégé — Nécessite un token d'accès HuggingFace. Courant pour les dépôts officiels de Qwen, meta-llama.
  • Ouvert (pas de badge) — Aucune authentification requise. Repacks communautaires de bartowski/*, lmstudio-community/*, TheBloke/*.

Processus de téléchargement

Lorsque vous cliquez sur Télécharger, GpuLLM effectue automatiquement :

  1. Vérifie le consentement de licence (silencieux pour les licences permissives ; affiche une boîte de dialogue pour les licences restreintes).
  2. Interroge l'API HuggingFace pour localiser le bon fichier GGUF.
  3. Télécharge avec prise en charge de la reprise HTTP (fichier temporaire .part).
  4. Vérifie le hachage SHA-256 par rapport aux métadonnées LFS HuggingFace.
Token requis ? Si un téléchargement échoue avec "Access Token Required", créez un token Read HuggingFace sur huggingface.co/settings/tokens et saisissez-le dans les Paramètres. Votre token est stocké dans le Gestionnaire d'informations d'identification Windows (chiffré DPAPI) et jamais journalisé ni écrit sur le disque.

Gestion des téléchargements

  • Annuler : Cliquez sur annuler sur une fiche en cours de téléchargement. Le fichier partiel .part est conservé pour une reprise ultérieure.
  • Changer d'emplacement : Allez dans Paramètres → Répertoire de téléchargement pour choisir un nouveau dossier.
  • Ouvrir le dossier : Sur une fiche téléchargée, cliquez sur l'icône de dossier pour ouvrir l'Explorateur à l'emplacement du fichier.
  • Importer GGUF : Utilisez le bouton "Importer un modèle" ou le glisser-déposer pour ajouter des fichiers GGUF existants.

Inférence et Benchmarking expliqués

Sélection du backend

GpuLLM détecte automatiquement votre matériel et sélectionne le backend le plus rapide disponible dans cet ordre : CUDA 12 → Vulkan → CPU. Aucune configuration manuelle requise — la chaîne de backend est affichée dans les résultats du benchmark après chaque inférence.

Stratégie de chargement des modèles

Lorsque vous cliquez sur Charger sur une fiche de modèle, les poids du modèle sont pré-chargés dans la mémoire GPU/CPU et restent résidents. Tous les appels d'inférence suivants réutilisent les poids pré-chargés, évitant complètement les E/S disque pour des temps de réponse rapides. Un seul modèle peut être chargé à la fois (deux en Mode Double).

Métriques de benchmark

Après chaque inférence, GpuLLM rapporte :

  • TTFT — Time-To-First-Token : délai avant l'apparition du premier token de réponse.
  • Tokens/s — Débit de génération après le premier token.
  • Latence — Temps de réponse total de bout en bout.
  • Pic GPU — Utilisation GPU maximale, utilisation VRAM et température pendant l'inférence.
  • Chaîne de backend — Quel backend a été utilisé (ex. "CUDA 12 → Vulkan → repli CPU").

Le rapport de session agrège les métriques de toutes les inférences depuis le lancement de l'application (tokens cumulés, latence moyenne, valeurs GPU de pointe).

Options d'exportation

  • Exporter PDF : Enregistre la conversation complète plus le rapport de benchmark de session en PDF A4.
  • Copier les messages : Chaque message a un bouton de copie pour l'exportation dans le presse-papiers.

Conseils et dépannage

Raccourcis clavier

  • Entrée ou Ctrl+Entrée — Envoyer le prompt dans Chat Benchmark.

Bonnes pratiques

  • Sécurité du token : Votre token HF est chiffré via Windows DPAPI et stocké dans le Gestionnaire d'informations d'identification — il n'est jamais journalisé dans des fichiers. Révoquez-le à tout moment depuis huggingface.co/settings/tokens.

Dépannage

  • Le téléchargement échoue avec 401/403 : Le modèle est protégé. Créez un token Read HuggingFace sur huggingface.co/settings/tokens et saisissez-le dans les Paramètres, puis réessayez.
  • Le téléchargement commence mais échoue avec "Échec de la vérification d'intégrité" : Le fichier a été corrompu pendant le transfert. Cliquez à nouveau sur Télécharger — GpuLLM recommence.
  • Limite de débit / HTTP 429 : HuggingFace limite l'accès API anonyme. Configurez un token pour des limites plus élevées.
  • Le modèle n'apparaît pas comme "Téléchargé" après redémarrage : Votre répertoire de téléchargement a peut-être changé. Vérifiez Paramètres → Répertoire de téléchargement ou utilisez le bouton Importer.
  • L'inférence est lente : Vérifiez que le backend CUDA est actif (affiché dans les résultats de benchmark). S'il repasse sur CPU, assurez-vous que votre pilote NVIDIA est à jour.
  • CUDA non détecté / Le backend repasse sur CPU : Installez ou mettez à jour votre pilote NVIDIA vers la version 545 ou ultérieure. Pour les GPU portables, assurez-vous que GpuLLM s'exécute sur le GPU dédié, pas sur le GPU intégré. Vérifiez les Paramètres graphiques Windows pour attribuer GpuLLM au GPU haute performance.
  • "Out of Memory" ou le modèle ne se charge pas : Votre GPU n'a pas assez de VRAM. Essayez un modèle plus petit ou une quantification plus agressive (ex. Q4_K_M au lieu de Q8_0). Utilisez Mon Matériel pour filtrer les modèles compatibles. Fermez les autres applications lourdes en GPU (navigateurs, jeux, éditeurs vidéo).
  • Le modèle se charge mais génère du charabia : Le fichier GGUF est peut-être corrompu ou au mauvais format. Vérifiez le hachage SHA-256 après le téléchargement. Assurez-vous que le modèle est au format GGUF (pas safetensors ou PyTorch).
  • Le Mode Double affiche le même modèle deux fois : Vous devez charger deux modèles différents avant d'activer le Mode Double. Chargez d'abord LLM1, puis LLM2 depuis la Bibliothèque de modèles.

Glossaire

GGUFUn format de fichier pour stocker des modèles de langage quantifiés, conçu par le projet llama.cpp. GpuLLM utilise exclusivement GGUF.
QuantificationTechnique qui réduit l'empreinte mémoire d'un modèle en abaissant la précision numérique (ex. de 16 bits à 4 bits). Les quantifications plus petites utilisent moins de VRAM mais peuvent avoir une qualité légèrement inférieure. Formats courants : Q8_0, Q4_K_M, Q2_K.
TTFTTime-To-First-Token — le délai entre l'envoi d'un prompt et la réception du premier mot de la réponse. Plus c'est bas, mieux c'est.
Tokens/sTokens générés par seconde après le premier token. C'est la principale mesure du débit d'inférence.
MMLUMassive Multitask Language Understanding — un benchmark standardisé couvrant 57 matières. GpuLLM inclut 100 questions anglaises échantillonnées.
C-EvalUn benchmark d'évaluation académique chinois. GpuLLM inclut 100 questions échantillonnées pour évaluer les capacités des modèles en langue chinoise.
HuggingFace LFSLarge File Storage — système basé sur Git pour stocker les fichiers modèles sur HuggingFace Hub. Chaque fichier a un hachage SHA-256 pour la vérification d'intégrité.
SHA-256Une fonction de hachage cryptographique utilisée pour vérifier que les fichiers modèles téléchargés sont complets et non altérés.
DPAPIWindows Data Protection API — chiffre les données sensibles (comme votre token HuggingFace) afin qu'elles ne puissent être déchiffrées que par le même compte utilisateur Windows.
VRAMVideo RAM — la mémoire dédiée de votre GPU. Le chargement et l'inférence des modèles consomment tous deux de la VRAM. Les modèles plus grands nécessitent plus de VRAM.
BackendLe moteur de calcul qui exécute l'inférence du modèle. GpuLLM prend en charge CUDA (GPU NVIDIA), Vulkan (GPU multi-fournisseurs) et CPU (repli).
