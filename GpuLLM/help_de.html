Was GpuLLM kann

GpuLLM ist eine Windows-Desktop-Anwendung, die Ihren PC in eine vollständige lokale KI-Workstation verwandelt. Alle Berechnungen laufen auf Ihrer Hardware — nichts verlässt Ihr Gerät.

🧠

LLMs lokal ausführen

Laden Sie GGUF-Modelle und chatten Sie direkt auf Ihrer GPU. Unterstützt CUDA-, Vulkan- und CPU-Backends mit automatischem Fallback.

📊

Hardware überwachen

Echtzeit-GPU-Auslastung, VRAM, Temperatur, Stromverbrauch, CPU-Auslastung und Arbeitsspeicher — alles in einem Dashboard mit 60-Sekunden-Sparkline-Diagrammen.

🔬

Modelle benchmarken

Messen Sie die Inferenzgeschwindigkeit (TTFT, Tokens/s, Latenz) und bewerten Sie die Modellgenauigkeit mit integrierten MMLU- und C-Eval-Fragesets.

📦

Modelle herunterladen & verwalten

Durchsuchen Sie einen kuratierten Katalog mit 13 Modellen von HuggingFace, laden Sie mit Resume-Unterstützung und SHA-256-Verifizierung herunter oder importieren Sie eigene GGUF-Dateien.

💬

Dual-Modell-Chat

Vergleichen Sie zwei Modelle nebeneinander in einer einzigen Konversation. LLM1 antwortet, LLM2 bewertet — beide sehen den vollständigen Kontext.

😺

Prism Cat Maskottchen

Ein lebendiger animierter GPU-Geist, der Farbe und Emotion basierend auf Ihrer GPU-Auslastung ändert.

100 % Lokal & Privat: Sobald Modelle heruntergeladen sind, ist keine Internetverbindung für die Inferenz erforderlich. Keine Telemetrie, keine Cloud-Abhängigkeit, kein Daten-Upload.

Schnellstart — Ersteinrichtung

Befolgen Sie diese Schritte, um von null zur ersten lokalen LLM-Inferenz zu gelangen.

  1. GpuLLM herunterladen & installieren

    Holen Sie sich GpuLLM aus dem Microsoft Store. Die App ist eigenständig — keine zusätzlichen Laufzeiten oder CUDA-Toolkit erforderlich.

  2. HuggingFace Access Token konfigurieren (optional)

    Öffnen Sie Einstellungen und geben Sie einen HuggingFace Read-Token ein. Dies wird nur für geschützte Modelle benötigt (z. B. Qwen, Meta-Llama). Community-Modelle von bartowski/* funktionieren ohne Token.

    So erstellen Sie einen Token: Gehen Sie zu huggingface.co/settings/tokens, klicken Sie auf New token, wählen Sie Typ Read, geben Sie einen Namen ein (z. B. "GpuLLM") und kopieren Sie den generierten Token. Fügen Sie ihn in die GpuLLM-Einstellungen ein. Der Token wird verschlüsselt im Windows Credential Manager (DPAPI) gespeichert und niemals protokolliert oder auf die Festplatte geschrieben.
  3. Modell herunterladen

    Navigieren Sie zur Modellbibliothek, durchsuchen Sie den kuratierten Katalog und klicken Sie auf Herunterladen auf einer beliebigen Modellkarte. Verwenden Sie die Schaltfläche Meine Hardware, um Modelle zu filtern, die mit dem VRAM Ihrer GPU kompatibel sind.

  4. Modell laden

    Sobald der Download abgeschlossen ist, klicken Sie auf Laden auf der Modellkarte. Die Modellgewichte werden in den GPU-Speicher geladen und bleiben für schnelle Inferenz resident.

  5. Chat starten

    Wechseln Sie zu Chat-Benchmark, geben Sie Ihren Prompt ein und drücken Sie die Eingabetaste. Das Modell streamt seine Antwort in Echtzeit, während die GPU-Metriken live in der Seitenleiste aktualisiert werden.

Profi-Tipp: Beginnen Sie mit einem kleineren Modell (z. B. Llama 3.2 1B oder DeepSeek-R1-Distill 1.5B), um zu überprüfen, ob alles funktioniert, bevor Sie zu größeren Modellen wechseln.

Häufige Anwendungsszenarien

Sie sind sich nicht sicher, wo Sie anfangen sollen? Hier sind einige typische Arbeitsabläufe zur Orientierung.

🎯 "Ich möchte das schnellste Modell für meine GPU finden"

Gehen Sie zur Modellbibliothek, klicken Sie auf Meine Hardware, um kompatible Modelle zu filtern, laden Sie einige herunter, laden Sie jedes und verwenden Sie Chat-Benchmark, um deren Tokens/s-Werte zu vergleichen. Überprüfen Sie die Benchmark-KPI-Karten nach jeder Inferenz.

🤝 "Ich möchte zwei Modelle nebeneinander vergleichen"

Laden Sie zwei Modelle, wechseln Sie zu Chat-Benchmark, aktivieren Sie den Dual-Modus und weisen Sie eines als Antwortgeber und das andere als Prüfer zu. Beide Modelle antworten auf denselben Prompt — vergleichen Sie die Antworten in Echtzeit.

📊 "Ich möchte wissen, wie genau mein Modell ist"

Laden Sie ein Modell, navigieren Sie zu Evaluierung, wählen Sie MMLU oder C-Eval und klicken Sie auf Start. Sie erhalten Genauigkeitswerte pro Kategorie und eine Aufschlüsselung nach Schwierigkeitsgrad — exportierbar als Markdown.

💰 "Ich möchte sehen, wie viel Geld ich im Vergleich zu Cloud-APIs spare"

Führen Sie eine beliebige Inferenz im Chat-Benchmark aus. Der integrierte Kostenschätzer vergleicht automatisch Ihre lokalen Inferenzkosten (Strom + Hardware-Abschreibung) mit den API-Preisen von OpenAI und Anthropic. Überprüfen Sie den Sitzungsbericht für kumulierte Einsparungen.

📥 "Ich habe bereits GGUF-Dateien — kann ich sie verwenden?"

Ja. Klicken Sie in der Modellbibliothek auf Modell importieren und wählen Sie Ihre GGUF-Datei aus, oder ziehen Sie sie einfach per Drag-and-Drop auf das Fenster. GpuLLM fügt sie Ihrer Bibliothek hinzu, ohne sie erneut herunterzuladen.

App-Navigation

GpuLLM hat fünf Hauptseiten, die über die linke Seitenleiste zugänglich sind. Klicken Sie auf ein Symbol, um die Seite zu wechseln.

Chat-Benchmark

Die Hauptseite für Inferenz und Benchmarking. Hier können Sie:

  • Mit einem Modell chatten: Geben Sie Text in das Feld Aufgabeneingabe ein und drücken Sie die Eingabetaste (oder Strg+Eingabe) zum Senden. Das Modell streamt seine Antwort Token für Token.
  • GPU-Metriken überwachen: Das rechte Panel zeigt Echtzeit-GPU-Auslastung, VRAM-Nutzung, Temperatur, Stromverbrauch, CPU-Auslastung und Systemspeicher — alle mit 60-Sekunden-Sparkline-Trends.
  • Inferenzparameter anpassen: Verwenden Sie die Schieberegler oben, um Max Tokens und Temperature für jede Anfrage einzustellen.
  • Benchmark-Ergebnisse anzeigen: Nach jeder Inferenz zeigen KPI-Karten TTFT (Time-To-First-Token), Tokens pro Sekunde, Latenz, Backend-Kette und GPU-Spitzenmetriken an.
  • Chat exportieren: Klicken Sie auf PDF exportieren, um die Konversation und den Benchmark-Bericht als A4-PDF-Dokument zu speichern.
  • Nachrichten kopieren: Jede Nachrichtenblase hat eine Kopierschaltfläche und einen Zeitstempel (HH:mm:ss).

Dual-Modell-Konversation

Aktivieren Sie den Dual-Modus im Chat-Benchmark, um die Relay-Inferenz mit zwei Modellen zu ermöglichen:

  • Wählen Sie ein primäres Modell (LLM1) und ein sekundäres Modell (LLM2) aus den Dropdown-Menüs.
  • LLM1 erhält Ihren Prompt und generiert eine Antwort.
  • LLM2 erhält automatisch den vollständigen Konversationskontext und liefert eine Überprüfung oder zweite Meinung.
  • Nachrichten sind nach Rolle gekennzeichnet: LLM1 · Antwortgeber und LLM2 · Prüfer.

Evaluierung

Führen Sie strukturierte Benchmarks gegen geladene Modelle mit standardisierten Fragesets durch:

  • MMLU: 100 ausgewählte Fragen aus 57 Fachgebieten (MINT, Geisteswissenschaften, Sozialwissenschaften).
  • C-Eval: 100 ausgewählte Fragen aus chinesischen akademischen Evaluierungsbenchmarks.
  • Die Seite hat drei Phasen: Konfigurieren (Modell + Benchmark wählen) → Ausführen (Echtzeit-Fortschritt) → Ergebnisse (Genauigkeit nach Kategorie und Schwierigkeitsgrad).
  • Eine Aufwärminferenz wird zuerst ausgeführt, um sicherzustellen, dass das Modell vollständig geladen ist, bevor die Bewertung beginnt.

Modellbibliothek

Durchsuchen, herunterladen und verwalten Sie Ihre GGUF-Modellsammlung:

  • Filter-Pills: Nach Kategorie eingrenzen — Coding, Chat oder Reasoning.
  • Meine Hardware: Ein-Klick-Filter, um nur Modelle anzuzeigen, die Ihre GPU ausführen kann. Kompatible Modelle zeigen einen grünen Haken.
  • Herunterladen: Klicken Sie auf die Download-Schaltfläche einer Karte. GpuLLM übernimmt automatisch Lizenzzustimmung, Dateiauflistung, Download mit Resume und SHA-256-Verifizierung.
  • Importieren: Per Drag-and-Drop oder Klick auf „Modell importieren", um beliebige vorhandene GGUF-Dateien hinzuzufügen.
  • Verwalten: Ordner öffnen, Pfad kopieren oder Modelle von ihren Karten laden/entladen.

Einstellungen

Konfigurieren Sie die Anwendung nach Ihren Wünschen:

  • Design: Wählen Sie Ihr bevorzugtes visuelles Design.
  • Download-Verzeichnis: Wählen Sie, wo Modelldateien gespeichert werden.
  • HuggingFace-Token: Geben Sie Ihren HuggingFace Access Token ein oder löschen Sie ihn (sicher im Windows Credential Manager gespeichert).
  • Rechtliche Links: Zugriff auf die Datenschutzerklärung und Drittanbieter-Hinweise.

Modell-Downloads verstehen

Geschützte vs. offene Modelle

Modellkarten zeigen ein Badge, das angibt, ob eine Authentifizierung erforderlich ist:

  • 🔒 Geschützt — Erfordert einen HuggingFace Access Token. Häufig bei offiziellen Repos von Qwen, meta-llama.
  • Offen (kein Badge) — Keine Authentifizierung erforderlich. Community-Repacks von bartowski/*, lmstudio-community/*, TheBloke/*.

Download-Prozess

Wenn Sie auf Herunterladen klicken, führt GpuLLM automatisch folgende Schritte aus:

  1. Prüft die Lizenzzustimmung (stumm bei freizügigen Lizenzen; zeigt Dialog bei eingeschränkten Lizenzen).
  2. Fragt die HuggingFace-API ab, um die korrekte GGUF-Datei zu finden.
  3. Lädt mit HTTP-Resume-Unterstützung herunter (.part temporäre Datei).
  4. Verifiziert den SHA-256-Hash anhand der HuggingFace-LFS-Metadaten.
Token erforderlich? Wenn ein Download mit „Access Token Required" fehlschlägt, erstellen Sie einen HuggingFace Read-Token unter huggingface.co/settings/tokens und geben Sie ihn in den Einstellungen ein. Ihr Token wird im Windows Credential Manager (DPAPI-verschlüsselt) gespeichert und niemals protokolliert oder auf die Festplatte geschrieben.

Downloads verwalten

  • Abbrechen: Klicken Sie auf Abbrechen auf einer herunterladenden Karte. Die partielle .part-Datei wird für spätere Wiederaufnahme beibehalten.
  • Speicherort ändern: Gehen Sie zu Einstellungen → Download-Verzeichnis, um einen neuen Ordner auszuwählen.
  • Ordner öffnen: Klicken Sie auf einer heruntergeladenen Karte auf das Ordnersymbol, um den Explorer am Dateispeicherort zu öffnen.
  • GGUF importieren: Verwenden Sie die Schaltfläche „Modell importieren" oder Drag-and-Drop, um vorhandene GGUF-Dateien hinzuzufügen.

Inferenz & Benchmarking erklärt

Backend-Auswahl

GpuLLM erkennt automatisch Ihre Hardware und wählt das schnellste verfügbare Backend in dieser Reihenfolge: CUDA 12 → Vulkan → CPU. Keine manuelle Konfiguration erforderlich — die Backend-Kette wird nach jeder Inferenz in den Benchmark-Ergebnissen angezeigt.

Modellladestrategie

Wenn Sie auf Laden auf einer Modellkarte klicken, werden die Modellgewichte in den GPU-/CPU-Speicher vorgeladen und bleiben resident. Alle nachfolgenden Inferenzaufrufe verwenden die vorgeladenen Gewichte wieder und überspringen den Festplatten-I/O vollständig für schnelle Antwortzeiten. Es kann nur ein Modell gleichzeitig geladen sein (zwei im Dual-Modus).

Benchmark-Metriken

Nach jeder Inferenz meldet GpuLLM:

  • TTFT — Time-To-First-Token: Wie lange es dauert, bis das erste Antwort-Token erscheint.
  • Tokens/s — Generierungsdurchsatz nach dem ersten Token.
  • Latenz — Gesamte End-to-End-Antwortzeit.
  • GPU-Spitze — Maximale GPU-Auslastung, VRAM-Nutzung und Temperatur während der Inferenz.
  • Backend-Kette — Welches Backend verwendet wurde (z. B. „CUDA 12 → Vulkan → CPU-Fallback").

Der Sitzungsbericht aggregiert Metriken über alle Inferenzen seit dem App-Start (kumulative Tokens, durchschnittliche Latenz, GPU-Spitzenwerte).

Exportoptionen

  • PDF exportieren: Speichert die vollständige Chat-Konversation plus den Sitzungs-Benchmark-Bericht als A4-PDF.
  • Nachrichten kopieren: Jede Nachricht hat eine Kopierschaltfläche für den Export in die Zwischenablage.

Tipps & Fehlerbehebung

Tastenkombinationen

  • Eingabe oder Strg+Eingabe — Prompt im Chat-Benchmark senden.

Bewährte Praktiken

  • Token-Sicherheit: Ihr HF-Token wird über Windows DPAPI verschlüsselt und im Credential Manager gespeichert — er wird niemals in Dateien protokolliert. Widerrufen Sie ihn jederzeit unter huggingface.co/settings/tokens.

Fehlerbehebung

  • Modell-Download schlägt mit 401/403 fehl: Das Modell ist geschützt. Erstellen Sie einen HuggingFace Read-Token unter huggingface.co/settings/tokens und geben Sie ihn in den Einstellungen ein, dann versuchen Sie es erneut.
  • Download beginnt, schlägt aber mit „Integritätsprüfung fehlgeschlagen" fehl: Die Datei wurde während der Übertragung beschädigt. Klicken Sie erneut auf Herunterladen — GpuLLM startet einen neuen Download.
  • Ratenbegrenzung / HTTP 429: HuggingFace begrenzt den anonymen API-Zugriff. Konfigurieren Sie einen Token für höhere Ratenlimits.
  • Modell wird nach Neustart nicht als „Heruntergeladen" angezeigt: Ihr Download-Verzeichnis hat sich möglicherweise geändert. Überprüfen Sie Einstellungen → Download-Verzeichnis oder verwenden Sie die Import-Schaltfläche.
  • Inferenz ist langsam: Überprüfen Sie, ob das CUDA-Backend aktiv ist (in den Benchmark-Ergebnissen angezeigt). Falls auf CPU zurückgegriffen wird, stellen Sie sicher, dass Ihr NVIDIA-Treiber aktuell ist.
  • CUDA nicht erkannt / Backend fällt auf CPU zurück: Installieren oder aktualisieren Sie Ihren NVIDIA-Treiber auf Version 545 oder höher. Stellen Sie bei Laptop-GPUs sicher, dass GpuLLM auf der dedizierten GPU läuft, nicht auf der integrierten. Überprüfen Sie die Windows-Grafikeinstellungen, um GpuLLM der Hochleistungs-GPU zuzuweisen.
  • „Out of Memory" oder Modell kann nicht geladen werden: Ihre GPU hat nicht genügend VRAM. Versuchen Sie ein kleineres Modell oder eine aggressivere Quantisierung (z. B. Q4_K_M statt Q8_0). Verwenden Sie Meine Hardware, um kompatible Modelle zu filtern. Schließen Sie andere GPU-intensive Anwendungen (Browser, Spiele, Video-Editoren).
  • Modell wird geladen, generiert aber Kauderwelsch: Die GGUF-Datei könnte beschädigt sein oder das falsche Format haben. Überprüfen Sie den SHA-256-Hash nach dem Download. Stellen Sie sicher, dass das Modell im GGUF-Format vorliegt (nicht safetensors oder PyTorch).
  • Dual-Modus zeigt dasselbe Modell zweimal an: Sie müssen zwei verschiedene Modelle laden, bevor Sie den Dual-Modus aktivieren. Laden Sie zuerst LLM1, dann LLM2 aus der Modellbibliothek.

Glossar

GGUFEin Dateiformat zum Speichern quantisierter Large Language Models, entwickelt vom llama.cpp-Projekt. GpuLLM verwendet ausschließlich GGUF.
QuantisierungTechnik, die den Speicherbedarf eines Modells durch Reduzierung der numerischen Genauigkeit verringert (z. B. von 16-Bit auf 4-Bit). Kleinere Quantisierungen verbrauchen weniger VRAM, können aber eine etwas geringere Qualität aufweisen. Gängige Formate: Q8_0, Q4_K_M, Q2_K.
TTFTTime-To-First-Token — die Verzögerung zwischen dem Senden eines Prompts und dem Empfang des ersten Wortes der Antwort. Niedriger ist besser.
Tokens/sNach dem ersten Token generierte Tokens pro Sekunde. Dies ist das primäre Maß für den Inferenzdurchsatz.
MMLUMassive Multitask Language Understanding — ein standardisierter Benchmark mit 57 Fachgebieten. GpuLLM enthält 100 ausgewählte englische Fragen.
C-EvalEin chinesischer akademischer Evaluierungsbenchmark. GpuLLM enthält 100 ausgewählte Fragen zur Bewertung der Fähigkeiten chinesischsprachiger Modelle.
HuggingFace LFSLarge File Storage — Git-basiertes System zum Speichern von Modelldateien auf dem HuggingFace Hub. Jede Datei hat einen SHA-256-Hash zur Integritätsprüfung.
SHA-256Eine kryptografische Hash-Funktion zur Überprüfung, ob heruntergeladene Modelldateien vollständig und unverändert sind.
DPAPIWindows Data Protection API — verschlüsselt sensible Daten (wie Ihren HuggingFace-Token), sodass sie nur vom selben Windows-Benutzerkonto entschlüsselt werden können.
VRAMVideo RAM — der dedizierte Speicher Ihrer GPU. Sowohl das Laden als auch die Inferenz von Modellen verbrauchen VRAM. Größere Modelle benötigen mehr VRAM.
BackendDie Berechnungs-Engine, die die Modellinferenz ausführt. GpuLLM unterstützt CUDA (NVIDIA-GPUs), Vulkan (plattformübergreifende GPUs) und CPU (Fallback).
