Возможности GpuLLM

GpuLLM — это настольное приложение Windows, которое превращает ваш ПК в полноценную локальную ИИ-рабочую станцию. Все вычисления выполняются на вашем оборудовании — ничего не покидает устройство.

🧠

Запуск LLM локально

Загружайте модели в формате GGUF и общайтесь прямо на вашей GPU. Поддержка бэкендов CUDA, Vulkan и CPU с автоматическим переключением.

📊

Мониторинг оборудования

Загрузка GPU в реальном времени, VRAM, температура, энергопотребление, загрузка CPU и память — всё на одной панели с 60-секундными спарклайн-графиками.

🔬

Тестирование моделей

Измеряйте скорость вывода (TTFT, токенов/с, задержка) и оценивайте точность моделей с помощью встроенных наборов вопросов MMLU и C-Eval.

📦

Загрузка и управление моделями

Просматривайте каталог из 13 моделей HuggingFace, скачивайте с поддержкой докачки и проверкой SHA-256 или импортируйте свои GGUF-файлы.

💬

Чат с двумя моделями

Сравнивайте две модели одновременно в одном разговоре. LLM1 отвечает, LLM2 проверяет — обе видят полный контекст.

😺

Маскот Prism Cat

Анимированный дух GPU, который меняет цвет и эмоции в зависимости от нагрузки на вашу видеокарту.

100% Локально и Приватно: После загрузки моделей интернет не требуется для вывода. Никакой телеметрии, облачной зависимости или передачи данных.

Быстрый старт — Первая настройка

Следуйте этим шагам, чтобы от нуля дойти до первого локального вывода LLM.

  1. Скачайте и установите GpuLLM

    Загрузите GpuLLM из Microsoft Store. Приложение автономно — не требует дополнительных сред выполнения или CUDA toolkit.

  2. Настройте токен доступа HuggingFace (опционально)

    Откройте Настройки и введите токен Read HuggingFace. Требуется только для защищённых моделей (например, Qwen, Meta-Llama). Модели сообщества от bartowski/* работают без токена.

    Как создать токен: Перейдите на huggingface.co/settings/tokens, нажмите New token, выберите тип Read, дайте имя (например, "GpuLLM") и скопируйте созданный токен. Вставьте его в Настройки GpuLLM. Токен хранится в зашифрованном виде в Windows Credential Manager (DPAPI) и никогда не записывается в логи или на диск.
  3. Скачайте модель

    Перейдите в Библиотеку моделей, просмотрите каталог и нажмите Скачать на карточке модели. Используйте кнопку Моё оборудование для фильтрации моделей, совместимых с VRAM вашей GPU.

  4. Загрузите модель

    После загрузки нажмите Загрузить на карточке модели. Веса модели загружаются в память GPU и остаются там для быстрого вывода.

  5. Начните общение

    Перейдите в Чат-бенчмарк, введите промпт и нажмите Enter. Модель выдаёт ответ в реальном времени, а метрики GPU обновляются на боковой панели.

Совет: Начните с небольшой модели (например, Llama 3.2 1B или DeepSeek-R1-Distill 1.5B), чтобы проверить работу, прежде чем переходить к более крупным.

Сценарии использования

Не знаете с чего начать? Вот несколько типичных рабочих процессов.

🎯 "Хочу найти самую быструю модель для моей GPU"

Перейдите в Библиотеку моделей, нажмите Моё оборудование для фильтрации совместимых моделей, скачайте несколько, загрузите каждую и используйте Чат-бенчмарк для сравнения показателей токенов/с. Проверяйте KPI-карточки после каждого вывода.

🤝 "Хочу сравнить две модели одновременно"

Загрузите две модели, перейдите в Чат-бенчмарк, включите Двойной режим и назначьте одну как Отвечающего, а другую как Проверяющего. Обе модели отвечают на один промпт — сравнивайте ответы в реальном времени.

📊 "Хочу узнать точность своей модели"

Загрузите модель, перейдите в Оценка, выберите MMLU или C-Eval и нажмите Старт. Вы получите оценки точности по категориям и разбивку по уровню сложности — экспортируется в Markdown.

💰 "Хочу увидеть экономию по сравнению с облачными API"

Выполните любой вывод в Чат-бенчмарке. Встроенный оценщик затрат автоматически сравнивает стоимость локального вывода (электричество + амортизация оборудования) с ценами API OpenAI и Anthropic. Проверьте отчёт сессии для накопленной экономии.

📥 "У меня уже есть GGUF-файлы — можно их использовать?"

Да. В Библиотеке моделей нажмите Импорт модели и выберите ваш GGUF-файл или просто перетащите его в окно. GpuLLM добавит его в библиотеку без повторной загрузки.

Навигация по приложению

GpuLLM имеет пять основных страниц, доступных с левой боковой панели. Нажмите на иконку для переключения.

Чат-бенчмарк

Основная страница вывода и тестирования. Здесь вы можете:

  • Общаться с моделью: Введите текст в поле Ввод задачи и нажмите Enter (или Ctrl+Enter) для отправки. Модель выводит ответ токен за токеном.
  • Отслеживать метрики GPU: Правая панель показывает загрузку GPU, использование VRAM, температуру, энергопотребление, загрузку CPU и системную память — с 60-секундными спарклайн-трендами.
  • Настраивать параметры вывода: Используйте ползунки вверху для установки Max Tokens и Temperature.
  • Просматривать результаты: После каждого вывода KPI-карточки показывают TTFT, токенов/с, задержку, цепочку бэкендов и пиковые метрики GPU.
  • Экспортировать чат: Нажмите Экспорт PDF для сохранения разговора и отчёта в PDF A4.
  • Копировать сообщения: Каждое сообщение имеет кнопку копирования и метку времени (ЧЧ:мм:сс).

Разговор с двумя моделями

Включите Двойной режим в Чат-бенчмарке для эстафетного вывода с двумя моделями:

  • Выберите основную модель (LLM1) и дополнительную (LLM2) из выпадающих списков.
  • LLM1 получает промпт и генерирует ответ.
  • LLM2 автоматически получает полный контекст разговора и предоставляет рецензию или второе мнение.
  • Сообщения помечены по ролям: LLM1 · Отвечающий и LLM2 · Проверяющий.

Оценка

Запуск структурированных тестов на загруженных моделях с использованием стандартизированных наборов вопросов:

  • MMLU: 100 вопросов по 57 предметам (STEM, гуманитарные, социальные науки).
  • C-Eval: 100 вопросов из китайских академических тестов.
  • Три фазы: Настройка (выбор модели + теста) → Выполнение (прогресс в реальном времени) → Результаты (точность по категориям и сложности).
  • Сначала выполняется разогревочный вывод для полной загрузки модели перед началом оценки.

Библиотека моделей

Просмотр, загрузка и управление коллекцией GGUF-моделей:

  • Фильтры: Сужение по категориям — Coding, Chat или Reasoning.
  • Моё оборудование: Фильтр в один клик для отображения только моделей, которые может запустить ваша GPU. Совместимые модели показывают зелёную галочку.
  • Скачать: Нажмите кнопку Скачать на карточке. GpuLLM автоматически обрабатывает согласие с лицензией, список файлов, загрузку с докачкой и проверку SHA-256.
  • Импорт: Перетащите или нажмите "Импорт модели" для добавления существующих GGUF-файлов.
  • Управление: Открыть папку, скопировать путь или загрузить/выгрузить модели.

Настройки

Настройте приложение под свои предпочтения:

  • Тема: Выберите визуальную тему.
  • Каталог загрузок: Выберите папку для хранения моделей.
  • Токен HuggingFace: Введите или удалите токен (безопасно хранится в Windows Credential Manager).
  • Юридические ссылки: Политика конфиденциальности и уведомления третьих сторон.

Загрузка моделей

Защищённые и открытые модели

Карточки моделей показывают значок, указывающий, требуется ли аутентификация:

  • 🔒 Защищённая — Требуется токен доступа HuggingFace. Характерно для официальных репозиториев Qwen, meta-llama.
  • Открытая (без значка) — Аутентификация не требуется. Перепаковки сообщества от bartowski/*, lmstudio-community/*, TheBloke/*.

Процесс загрузки

При нажатии Скачать GpuLLM автоматически:

  1. Проверяет согласие с лицензией (без уведомлений для разрешительных лицензий; показывает диалог для ограниченных).
  2. Запрашивает API HuggingFace для поиска правильного GGUF-файла.
  3. Загружает с поддержкой докачки HTTP (временный файл .part).
  4. Проверяет хеш SHA-256 по метаданным LFS HuggingFace.
Нужен токен? Если загрузка не удалась с "Access Token Required", создайте токен Read HuggingFace на huggingface.co/settings/tokens и введите его в Настройках. Токен хранится в Windows Credential Manager (шифрование DPAPI) и никогда не логируется и не записывается на диск.

Управление загрузками

  • Отмена: Нажмите отмену на загружаемой карточке. Частичный файл .part сохраняется для последующей докачки.
  • Сменить расположение: Настройки → Каталог загрузок для выбора новой папки.
  • Открыть папку: На загруженной карточке нажмите значок папки для открытия в Проводнике.
  • Импорт GGUF: Используйте "Импорт модели" или перетаскивание для добавления GGUF-файлов.

Вывод и тестирование

Выбор бэкенда

GpuLLM автоматически определяет оборудование и выбирает самый быстрый доступный бэкенд в порядке: CUDA 12 → Vulkan → CPU. Ручная настройка не требуется — цепочка бэкендов отображается в результатах теста после каждого вывода.

Стратегия загрузки модели

При нажатии Загрузить на карточке модели веса предварительно загружаются в память GPU/CPU и остаются там. Все последующие выводы используют предзагруженные веса, полностью пропуская дисковый ввод-вывод для быстрого отклика. Одновременно может быть загружена только одна модель (две в Двойном режиме).

Метрики тестирования

После каждого вывода GpuLLM сообщает:

  • TTFT — Time-To-First-Token: время до первого токена ответа.
  • Токенов/с — Пропускная способность генерации после первого токена.
  • Задержка — Полное время ответа от начала до конца.
  • Пик GPU — Максимальная загрузка GPU, использование VRAM и температура во время вывода.
  • Цепочка бэкендов — Какой бэкенд использовался (например, "CUDA 12 → Vulkan → CPU запасной").

Отчёт сессии агрегирует метрики по всем выводам с момента запуска (суммарные токены, средняя задержка, пиковые значения GPU).

Экспорт

  • Экспорт PDF: Сохраняет полный чат и отчёт сессии как PDF A4.
  • Копирование сообщений: Каждое сообщение имеет кнопку копирования в буфер обмена.

Советы и устранение неполадок

Горячие клавиши

  • Enter или Ctrl+Enter — Отправить промпт в Чат-бенчмарке.

Лучшие практики

  • Безопасность токена: Токен HF шифруется через Windows DPAPI и хранится в Credential Manager — никогда не записывается в файлы. Отзовите его в любое время на huggingface.co/settings/tokens.

Устранение неполадок

  • Ошибка 401/403 при загрузке: Модель защищена. Создайте токен Read HuggingFace на huggingface.co/settings/tokens и введите в Настройках, затем повторите.
  • Ошибка проверки целостности: Файл повреждён при передаче. Нажмите Скачать ещё раз — GpuLLM начнёт заново.
  • Ограничение скорости / HTTP 429: HuggingFace ограничивает анонимный доступ. Настройте токен для повышения лимитов.
  • Модель не отображается как загруженная после перезапуска: Возможно, изменился каталог загрузок. Проверьте Настройки → Каталог загрузок или используйте Импорт.
  • Медленный вывод: Проверьте, активен ли бэкенд CUDA в результатах. Если используется CPU, обновите драйвер NVIDIA.
  • CUDA не обнаружена / используется CPU: Установите или обновите драйвер NVIDIA до версии 545 или новее. Для ноутбуков убедитесь, что GpuLLM работает на выделенной GPU. Проверьте Настройки графики Windows.
  • "Out of Memory" или модель не загружается: Недостаточно VRAM. Попробуйте модель меньшего размера или более агрессивную квантизацию (например, Q4_K_M вместо Q8_0). Используйте Моё оборудование для фильтрации. Закройте другие приложения, использующие GPU.
  • Модель загружается, но выдаёт бессвязный текст: Файл GGUF повреждён или неверного формата. Проверьте хеш SHA-256. Убедитесь в формате GGUF (не safetensors или PyTorch).
  • Двойной режим показывает одну модель дважды: Загрузите две разные модели перед включением Двойного режима. Сначала LLM1, затем LLM2 из Библиотеки моделей.

Глоссарий

GGUFФормат файла для хранения квантованных больших языковых моделей, разработанный проектом llama.cpp. GpuLLM использует исключительно GGUF.
КвантизацияТехника уменьшения потребления памяти моделью за счёт снижения числовой точности (например, с 16 до 4 бит). Меньшая квантизация использует меньше VRAM, но может незначительно снизить качество. Форматы: Q8_0, Q4_K_M, Q2_K.
TTFTTime-To-First-Token — задержка между отправкой промпта и получением первого слова ответа. Чем ниже, тем лучше.
Токенов/сТокенов в секунду после первого токена. Основная мера пропускной способности вывода.
MMLUMassive Multitask Language Understanding — стандартизированный тест по 57 предметам. GpuLLM включает 100 вопросов на английском.
C-EvalКитайский академический тест. GpuLLM включает 100 вопросов для оценки способностей моделей на китайском языке.
HuggingFace LFSLarge File Storage — система на основе Git для хранения файлов моделей на HuggingFace Hub. Каждый файл имеет хеш SHA-256 для проверки целостности.
SHA-256Криптографическая хеш-функция для проверки полноты и неизменности загруженных моделей.
DPAPIWindows Data Protection API — шифрует конфиденциальные данные (например, токен HuggingFace), доступные только той же учётной записи Windows.
VRAMВидеопамять — выделенная память GPU. Загрузка и вывод модели потребляют VRAM. Большие модели требуют больше VRAM.
БэкендВычислительный движок для вывода модели. GpuLLM поддерживает CUDA (NVIDIA), Vulkan (разные GPU) и CPU (запасной).
