GpuLLM Setup Guide

How to obtain a HuggingFace Access Token and download models for local LLM inference.

Part 1 — Obtaining a HuggingFace Access Token

ℹ Third-Party Service: By creating a HuggingFace account and access token, you agree to HuggingFace's Terms of Service and Privacy Policy. GpuLLM is not affiliated with HuggingFace, Inc.

Many official model repositories on HuggingFace (such as Qwen/*, microsoft/*, meta-llama/*) are gated — they require authentication before you can download files. GpuLLM uses a read-only HuggingFace Access Token to authenticate these requests. The token is stored securely in your Windows Credential Manager and is never transmitted to anyone other than HuggingFace.

ℹ Note: Open community repositories (such as those repackaged by bartowski/* or MaziyarPanahi/*) do not require a token for downloading GGUF files. A token is only needed when you see the orange "Gated" lock icon on a model card, or when a download fails with an "Access Token Required" dialog.

Step-by-Step: Create Your Token

  1. Sign up or log in at HuggingFace

    Visit huggingface.co/join to create a free account, or sign in if you already have one.

  2. Open Settings → Access Tokens

    Click your profile picture in the top-right corner, select Settings, then navigate to Access Tokens in the left sidebar. Or go directly to huggingface.co/settings/tokens.

  3. Create a new token

    Click the "Create new token" button. Choose the "Read" type (also called "Fine-grained" in newer HuggingFace versions) — this is the minimum permission required for downloading models.

    Tip: A Read token can only read and download public or gated repositories. It cannot modify, delete, or create anything. This is the safest choice.
  4. Set a descriptive name and click "Generate a token"

    Name your token something memorable like GpuLLM so you can identify it later. Copy the generated token immediately — you won't be able to see it again after you close the page.

    ⚠ Important: HuggingFace only shows the token once. After you navigate away or close the dialog, the token value cannot be retrieved again. If you lose it, you'll need to create a new token.
  5. Enter the token in GpuLLM Settings

    Open GpuLLM, navigate to the Settings page (gear icon in the left sidebar). Paste your token into the "HuggingFace Access Token" password field and click Save.

    The token is saved to Windows Credential Manager under the key GpuLLM:HuggingFaceToken. It is encrypted using DPAPI and never written to any file or log.

Want to verify? After saving your token, try downloading a gated model such as Qwen2.5-Coder 3B or Phi-4-mini. If the download starts successfully, your token is working correctly.

Part 2 — Downloading Models

GpuLLM includes a built-in Model Library with a growing collection of curated models across categories like Coding, NLP, Vision, and Science. Models come from official HuggingFace repositories as well as community-repacked GGUF variants.

Understanding Gated vs. Open Models

Each model card shows a badge indicating whether authentication is required:

  • 🔓 Gated (orange lock icon) — The HuggingFace repository requires a valid Access Token. This is common for official repositories from Qwen, microsoft, meta-llama, etc. Configure your token in Settings before downloading.
  • Open (no badge) — No authentication needed. Community repacks from bartowski/*, MaziyarPanahi/*, and others host open GGUF files that can be downloaded without a token.

How to Download a Model

  1. Navigate to Model Library

    Open GpuLLM and click the Model Library tab in the left navigation bar (box icon).

  2. Filter and find your model

    Use the filter pills at the top to narrow by capability category (Coding, NLP, Vision, Audio, Science) and VRAM requirement (4 GB, 8 GB, 16 GB, 24 GB, 32 GB+).

    Tip: Click the "My Hardware" button to instantly show only models that your GPU can run. GpuLLM automatically detects your GPU's VRAM and filters models accordingly. Compatible cards show a green checkmark.
  3. Click the model card

    Each card shows the model name, author, size, quantization type, and whether it requires authentication. Click the card to see additional details (parameter count, HuggingFace repository link, license info) in a detail popup.

  4. Start the download

    Click the Download button on the card (downward arrow icon). The following happens automatically:

    • License consent check: For permissive licenses (Apache-2.0, MIT), GpuLLM records consent silently and proceeds. For licenses with risk notes (Llama Community, Gemma Terms), a license agreement dialog is shown with the risk summary highlighted.
    • File listing: GpuLLM queries the HuggingFace API to find the correct GGUF file. If the API requires authentication and no token is configured, the app automatically falls back to the direct CDN download URL (which works for open repositories).
    • Download with resume: The file is downloaded using HTTP with support for resuming partial downloads. A .part temporary file holds partially downloaded data.
    • SHA-256 verification: After the download completes, GpuLLM verifies the file's integrity against the SHA-256 hash from HuggingFace's LFS metadata. If the check fails, the file is deleted and you are asked to retry.
  5. Monitor progress

    During download, the card updates in real time with a progress bar, current speed (e.g. "3.1 MB/s"), estimated time remaining, and downloaded/total size. You can cancel the download at any time by clicking the cancel button.

  6. Load the model

    Once downloaded, the card changes to "Downloaded" state. Click Load to load the model into the inference engine. The model weights are pre-loaded into GPU/CPU memory and stay resident, so subsequent inference calls skip disk I/O entirely. You can then use the model in the Chat Benchmark page.

Download Failure: Access Token Required

If a download fails because the repository requires authentication and you haven't configured a token (or your token has expired), GpuLLM shows a dedicated "Access Token Required" dialog. This happens when:

  • The HuggingFace API returns 401/403 (no auth or insufficient permissions).
  • The CDN fallback also returns 401/403 (the model is truly gated at both levels).

The dialog provides a link to this setup guide. Follow the steps in Part 1 above to create and configure your token, then retry the download.

Managing Your Downloads

  • Download directory: Change the default download location in Settings → Download Directory. All model files are downloaded to this folder.
  • Open folder: On a downloaded card, click the folder button to open Explorer directly at the model file location.
  • Copy path: Click the copy button to copy the full local file path to your clipboard.
  • Cancel downloads: Click the cancel button on a downloading card at any time. The partial .part file is retained so you can resume later.
  • Import local GGUF: In the Model Library, click "Import Model" to browse and add any GGUF file you already have on your disk.
⚠ Token security: Your HuggingFace token is stored in Windows Credential Manager (DPAPI-encrypted) and is never logged or written to disk. To remove your token, go to Settings and click the clear/delete button next to the token field. You can also revoke the token from the HuggingFace website at any time.

Troubleshooting

Download starts but fails with "Integrity check failed"

The downloaded file's SHA-256 hash does not match the expected value from HuggingFace. The partial file is automatically deleted. This is usually caused by an interrupted or corrupted transfer. Simply click Download again — GpuLLM will start fresh.

Download succeeds but model doesn't appear as "Downloaded" on next launch

If your download directory changed between sessions, or the file was moved/deleted, the card will reset to "Not Downloaded." Check Settings → Download Directory to confirm the path matches your actual model location. Alternatively, use the Import button to add the GGUF file at its current location.

"Access Token Required" dialog appears for a non-gated model

GpuLLM uses a two-tier download strategy: first the HuggingFace API (which may require authentication even for some open repos), then a direct CDN fallback. If both tiers fail with 401/403, the dialog appears. This can happen if HuggingFace temporarily restricts anonymous access. Wait a few minutes and retry, or configure a Read token to avoid rate limits.

Rate limit / HTTP 429 errors

HuggingFace imposes rate limits on anonymous API access. If you encounter frequent 429 responses, configure an Access Token — authenticated requests have higher rate limits and are less likely to be throttled. The HuggingFace API client in GpuLLM includes three automatic retries with exponential backoff (2s → 4s → 8s) for rate-limit handling.
