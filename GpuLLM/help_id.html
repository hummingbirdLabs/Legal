Kemampuan GpuLLM

GpuLLM adalah aplikasi desktop Windows yang mengubah PC Anda menjadi workstation AI lokal lengkap. Semua komputasi berjalan di hardware Anda — tidak ada yang meninggalkan perangkat.

🧠

Jalankan LLM Secara Lokal

Muat model format GGUF dan mengobrol langsung di GPU. Mendukung backend CUDA, Vulkan, dan CPU.

📊

Pantau Hardware

Utilisasi GPU real-time, VRAM, suhu, daya, CPU, dan memori — dalam satu dashboard.

🔬

Benchmark Model

Ukur kecepatan inferensi (TTFT, token/detik, latensi) dan evaluasi akurasi model dengan MMLU dan C-Eval.

📦

Unduh & Kelola Model

Jelajahi katalog 13 model dari HuggingFace, unduh dengan resume dan verifikasi SHA-256.

💬

Chat Dua Model

Bandingkan dua model berdampingan. LLM1 menjawab, LLM2 meninjau — keduanya melihat konteks penuh.

😺

Maskot Prism Cat

Animasi GPU spirit langsung yang berubah warna dan emosi berdasarkan beban kerja GPU.

100% Lokal & Privat: Setelah model diunduh, tidak perlu koneksi internet untuk inferensi. Tanpa telemetri, tanpa cloud.

Mulai Cepat — Pengaturan Pertama

Ikuti langkah-langkah ini dari nol hingga menjalankan inferensi LLM lokal pertama Anda.

  1. Unduh & instal GpuLLM

    Dapatkan GpuLLM dari Microsoft Store. Aplikasi mandiri — tidak perlu runtime tambahan atau CUDA toolkit.

  2. Konfigurasi Token Akses HuggingFace (opsional)

    Buka Pengaturan dan masukkan token Read HuggingFace. Hanya diperlukan untuk model berpagar (mis. Qwen, Meta-Llama).

  3. Unduh model

    Navigasi ke Pustaka Model, jelajahi katalog, dan klik Unduh pada kartu model. Gunakan Perangkat Saya untuk memfilter model yang kompatibel.

  4. Muat model

    Setelah diunduh, klik Muat pada kartu model. Bobot model dimuat ke memori GPU.

  5. Mulai mengobrol

    Beralih ke Chat Benchmark, ketik prompt, dan tekan Enter.

Tips pro: Mulai dengan model kecil (mis. Llama 3.2 1B atau DeepSeek-R1-Distill 1.5B) untuk verifikasi sebelum beralih ke model lebih besar.

Skenario Penggunaan Umum

🎯 "Saya ingin menemukan model tercepat untuk GPU saya"

Buka Pustaka Model, klik Perangkat Saya untuk filter, unduh beberapa, muat masing-masing, dan gunakan Chat Benchmark untuk membandingkan token/detik.

🤝 "Saya ingin membandingkan dua model berdampingan"

Muat dua model, beralih ke Chat Benchmark, aktifkan Dual Mode.

📊 "Saya ingin tahu seberapa akurat model saya"

Muat model, navigasi ke Evaluasi, pilih MMLU atau C-Eval, klik Mulai.

💰 "Saya ingin melihat penghematan vs API cloud"

Jalankan inferensi di Chat Benchmark. Estimator biaya otomatis membandingkan biaya lokal vs API OpenAI dan Anthropic.

📥 "Saya sudah punya file GGUF — bisakah digunakan?"

Ya. Di Pustaka Model, klik Impor Model atau drag-and-drop file GGUF Anda.

Navigasi Aplikasi

GpuLLM memiliki lima halaman utama yang dapat diakses dari sidebar kiri.

Chat Benchmark

Halaman inferensi dan benchmark utama. Di sini Anda dapat:

  • Mengobrol dengan model: Ketik di kotak Task Input dan tekan Enter.
  • Pantau metrik GPU: Panel kanan menampilkan metrik GPU real-time.
  • Sesuaikan parameter: Gunakan slider untuk Max Tokens dan Temperature.
  • Lihat hasil benchmark: Kartu KPI menunjukkan TTFT, token/detik, latensi.
  • Ekspor chat: Klik Ekspor PDF untuk menyimpan percakapan dan laporan.

Percakapan Dua Model

Aktifkan Dual Mode untuk inferensi dua model:

  • Pilih model utama (LLM1) dan model kedua (LLM2).
  • LLM1 menerima prompt dan menghasilkan jawaban.
  • LLM2 menerima konteks penuh dan memberikan tinjauan.

Evaluasi

Jalankan benchmark terstruktur dengan set soal standar:

  • MMLU: 100 soal dari 57 mata pelajaran.
  • C-Eval: 100 soal dari benchmark evaluasi akademik Tiongkok.

Pustaka Model

Jelajahi, unduh, dan kelola koleksi model GGUF:

  • Filter: Coding, Chat, atau Reasoning.
  • Perangkat Saya: Filter satu klik untuk model yang kompatibel dengan VRAM.
  • Unduh: Klik tombol Unduh pada kartu model.
  • Impor: Drag-and-drop atau klik "Impor Model".

Pengaturan

Konfigurasi aplikasi sesuai preferensi:

  • Tema: Pilih tema visual.
  • Direktori Unduhan: Pilih lokasi penyimpanan model.
  • Token HuggingFace: Masukkan atau hapus token akses.

Memahami Unduhan Model

Model Berpagar vs. Terbuka

  • 🔒 Berpagar — Memerlukan Token Akses HuggingFace.
  • Terbuka — Tidak perlu autentikasi.

Proses Unduhan

Saat Anda mengklik Unduh, GpuLLM otomatis:

  1. Memeriksa persetujuan lisensi.
  2. Mencari file GGUF yang benar via API HuggingFace.
  3. Mengunduh dengan dukungan resume HTTP.
  4. Memverifikasi hash SHA-256.
Perlu token? Jika unduhan gagal dengan "Access Token Required," buat token Read HuggingFace di huggingface.co/settings/tokens dan masukkan di Pengaturan.

Inferensi & Benchmark Dijelaskan

Pemilihan Backend

GpuLLM otomatis mendeteksi hardware dan memilih backend tercepat: CUDA 12 → Vulkan → CPU.

Metrik Benchmark

  • TTFT — Time-To-First-Token.
  • Token/detik — Throughput generasi.
  • Latensi — Total waktu respons.
  • Puncak GPU — Utilisasi, VRAM, suhu maksimum.
  • Rantai Backend — Backend yang digunakan.

Opsi Ekspor

  • Ekspor PDF: Simpan percakapan lengkap + laporan benchmark sebagai PDF A4.
  • Salin pesan: Setiap pesan memiliki tombol salin.

Tips & Troubleshooting

Praktik Terbaik

  • Mulai kecil: Uji dengan model 1-3B dulu.
  • Periksa VRAM: Gunakan filter Perangkat Saya.
  • Penyetelan suhu: Lebih rendah (0,1-0,5) untuk faktual/coding; lebih tinggi (0,7-1,0) untuk kreatif.
  • Max Tokens: 512-2048 untuk kebanyakan tugas.
  • Keamanan token: Token HF dienkripsi via DPAPI Windows.

Troubleshooting

  • Unduhan gagal 401/403: Model berpagar. Buat token Read HuggingFace.
  • Gagal pemeriksaan integritas: File rusak. Klik Unduh lagi.
  • Batas laju HTTP 429: Konfigurasikan token.
  • Inferensi lambat: Verifikasi backend CUDA aktif.
  • CUDA tidak terdeteksi: Perbarui driver NVIDIA ke versi 545+.
  • "Out of Memory": Coba model lebih kecil. Gunakan filter Perangkat Saya.
  • Model menghasilkan teks acak: Verifikasi hash SHA-256.
  • Dual Mode menampilkan model sama: Muat dua model berbeda dulu.

Glosarium

GGUFFormat file untuk menyimpan model LLM terkuantisasi, dirancang oleh proyek llama.cpp.
KuantisasiTeknik mengurangi jejak memori model dengan menurunkan presisi numerik.
TTFTTime-To-First-Token — jeda antara mengirim prompt dan menerima kata pertama respons.
Token/detikToken yang dihasilkan per detik setelah token pertama.
MMLUMassive Multitask Language Understanding — benchmark standar mencakup 57 mata pelajaran.
C-EvalBenchmark evaluasi akademik Tiongkok.
DPAPIWindows Data Protection API — mengenkripsi data sensitif sehingga hanya dapat didekripsi oleh akun pengguna Windows yang sama.
VRAMVideo RAM — memori khusus pada GPU Anda.
BackendMesin komputasi yang menjalankan inferensi model: CUDA, Vulkan, atau CPU.
