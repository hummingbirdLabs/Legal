GpuLLM क्या कर सकता है

GpuLLM एक विंडोज डेस्कटॉप एप्लिकेशन है जो आपके PC को एक संपूर्ण स्थानीय AI वर्कस्टेशन में बदल देता है। सभी गणना आपके हार्डवेयर पर चलती है — कुछ भी आपके डिवाइस से बाहर नहीं जाता।

🧠

LLM स्थानीय रूप से चलाएं

GGUF-प्रारूप मॉडल लोड करें और सीधे अपने GPU पर चैट करें। CUDA, Vulkan और CPU बैकएंड का समर्थन करता है।

📊

हार्डवेयर की निगरानी करें

रीयल-टाइम GPU उपयोग, VRAM, तापमान, बिजली खपत, CPU उपयोग और मेमोरी — सब एक डैशबोर्ड में 60-सेकंड स्पार्कलाइन चार्ट के साथ।

🔬

मॉडल बेंचमार्क करें

अनुमान गति (TTFT, tokens/s, विलंबता) मापें और अंतर्निहित MMLU और C-Eval प्रश्न सेट के साथ मॉडल सटीकता का मूल्यांकन करें।

📦

मॉडल डाउनलोड और प्रबंधित करें

HuggingFace से 13 मॉडलों का क्यूरेटेड कैटलॉग ब्राउज़ करें, रिज़्यूम सपोर्ट और SHA-256 सत्यापन के साथ डाउनलोड करें।

💬

डुअल-मॉडल चैट

एक ही बातचीत में दो मॉडलों की साथ-साथ तुलना करें। LLM1 उत्तर देता है, LLM2 समीक्षा करता है।

😺

प्रिज़्म कैट मैस्कट

एक जीवंत एनिमेटेड GPU आत्मा जो आपके GPU कार्यभार के आधार पर रंग और भावना बदलती है।

100% स्थानीय और निजी: मॉडल डाउनलोड होने के बाद, अनुमान के लिए इंटरनेट कनेक्शन की आवश्यकता नहीं है। कोई टेलीमेट्री नहीं, कोई क्लाउड निर्भरता नहीं।

सिस्टम आवश्यकताएं

इंस्टॉल करने से पहले, सुनिश्चित करें कि आपका PC इन आवश्यकताओं को पूरा करता है।

GPUNVIDIA GTX 10 श्रृंखला या नया (CUDA 12), या कोई Vulkan-संगत GPU
VRAMछोटे मॉडलों के लिए न्यूनतम 4 GB (1-3B); 8 GB अनुशंसित; 7B+ मॉडलों के लिए 16+ GB

स्टोरेजप्रति मॉडल फ़ाइल 1-20 GB; तेज़ मॉडल लोडिंग के लिए SSD अनुशंसित
इंटरनेटकेवल मॉडल डाउनलोड करने के लिए आवश्यक; सभी अनुमान ऑफ़लाइन चलते हैं
शक्तिशाली GPU नहीं है? GpuLLM केवल CPU सिस्टम पर भी काम करता है — बस धीमे अनुमान की अपेक्षा करें। Llama 3.2 1B या DeepSeek-R1-Distill 1.5B जैसे 1-3B मॉडल से शुरू करें।

त्वरित आरंभ — पहली बार सेटअप

शून्य से अपना पहला स्थानीय LLM अनुमान चलाने के लिए इन चरणों का पालन करें।

  1. GpuLLM डाउनलोड और इंस्टॉल करें

    Microsoft Store से GpuLLM प्राप्त करें। ऐप स्व-निहित है — किसी अतिरिक्त रनटाइम या CUDA टूलकिट की आवश्यकता नहीं है।

  2. अपना HuggingFace एक्सेस टोकन कॉन्फ़िगर करें (वैकल्पिक)

    सेटिंग्स खोलें और HuggingFace Read टोकन दर्ज करें। केवल सुरक्षित मॉडलों के लिए आवश्यक (जैसे Qwen, Meta-Llama)। bartowski/* के सामुदायिक मॉडल बिना टोकन के काम करते हैं।

    टोकन कैसे बनाएं: huggingface.co/settings/tokens पर जाएं, New token पर क्लिक करें, प्रकार Read चुनें, एक नाम दें (जैसे "GpuLLM") और जनरेट किए गए टोकन को कॉपी करें। इसे GpuLLM सेटिंग्स में पेस्ट करें। टोकन Windows Credential Manager (DPAPI) में एन्क्रिप्टेड स्टोर होता है।
  3. एक मॉडल डाउनलोड करें

    मॉडल लाइब्रेरी पर जाएं, क्यूरेटेड कैटलॉग ब्राउज़ करें और किसी भी मॉडल कार्ड पर Download पर क्लिक करें। अपने GPU के VRAM के साथ संगत मॉडल फ़िल्टर करने के लिए My Hardware बटन का उपयोग करें।

  4. मॉडल लोड करें

    डाउनलोड होने के बाद, मॉडल कार्ड पर Load पर क्लिक करें। मॉडल वेट GPU मेमोरी में लोड हो जाते हैं और तेज़ अनुमान के लिए रेसिडेंट रहते हैं।

  5. चैट शुरू करें

    Chat Benchmark पर स्विच करें, अपना प्रॉम्प्ट टाइप करें और Enter दबाएं। मॉडल रीयल-टाइम में अपनी प्रतिक्रिया स्ट्रीम करता है जबकि GPU मेट्रिक्स साइडबार में लाइव अपडेट होते हैं।

प्रो टिप: बड़े मॉडलों पर जाने से पहले सब कुछ काम कर रहा है यह सत्यापित करने के लिए एक छोटे मॉडल (जैसे Llama 3.2 1B या DeepSeek-R1-Distill 1.5B) से शुरू करें।

सामान्य उपयोग परिदृश्य

निश्चित नहीं कि कहां से शुरू करें? आपको मार्गदर्शन करने के लिए यहां कुछ विशिष्ट वर्कफ़्लो हैं।

🎯 "मैं अपने GPU के लिए सबसे तेज़ मॉडल खोजना चाहता हूं"

मॉडल लाइब्रेरी पर जाएं, संगत मॉडल फ़िल्टर करने के लिए My Hardware पर क्लिक करें, कुछ डाउनलोड करें, प्रत्येक को लोड करें और उनके tokens/s स्कोर की तुलना करने के लिए Chat Benchmark का उपयोग करें।

🤝 "मैं दो मॉडलों की साथ-साथ तुलना करना चाहता हूं"

दो मॉडल लोड करें, Chat Benchmark पर स्विच करें, Dual Mode टॉगल करें और एक को उत्तरदाता और दूसरे को समीक्षक के रूप में असाइन करें।

📊 "मैं जानना चाहता हूं कि मेरा मॉडल कितना सटीक है"

एक मॉडल लोड करें, Evaluation पर जाएं, MMLU या C-Eval चुनें और Start पर क्लिक करें।

💰 "मैं देखना चाहता हूं कि क्लाउड API की तुलना में मैं कितना पैसा बचाता हूं"

Chat Benchmark में कोई भी अनुमान चलाएं। अंतर्निहित लागत अनुमानक स्वचालित रूप से आपकी स्थानीय अनुमान लागत की तुलना OpenAI और Anthropic API मूल्य निर्धारण से करता है।

📥 "मेरे पास पहले से GGUF फ़ाइलें हैं — क्या मैं उनका उपयोग कर सकता हूं?"

हां। मॉडल लाइब्रेरी में, Import Model पर क्लिक करें और अपनी GGUF फ़ाइल चुनें, या बस इसे विंडो पर ड्रैग-एंड-ड्रॉप करें।

ऐप नेविगेशन

GpuLLM में बाएं साइडबार से पांच मुख्य पेज एक्सेस किए जा सकते हैं। पेज स्विच करने के लिए किसी भी आइकन पर क्लिक करें।

Chat Benchmark

मुख्य अनुमान और बेंचमार्किंग पेज। यहां आप कर सकते हैं:

  • मॉडल के साथ चैट करें: Task Input बॉक्स में टाइप करें और भेजने के लिए Enter (या Ctrl+Enter) दबाएं।
  • GPU मेट्रिक्स की निगरानी करें: दायां पैनल रीयल-टाइम GPU उपयोग, VRAM, तापमान, बिजली खपत दिखाता है।
  • अनुमान पैरामीटर समायोजित करें: Max Tokens और Temperature सेट करने के लिए स्लाइडर का उपयोग करें।
  • बेंचमार्क परिणाम देखें: KPI कार्ड TTFT, tokens/s, विलंबता दिखाते हैं।
  • चैट एक्सपोर्ट करें: Export PDF पर क्लिक करें।

डुअल-मॉडल कन्वर्सेशन

Chat Benchmark में Dual Mode टॉगल करें: LLM1 उत्तर देता है, LLM2 समीक्षा करता है।

Evaluation

MMLU और C-Eval का उपयोग करके संरचित बेंचमार्क चलाएं।

मॉडल डाउनलोड को समझना

सुरक्षित बनाम खुले मॉडल

  • 🔒 सुरक्षित — HuggingFace एक्सेस टोकन की आवश्यकता है।
  • खुला — कोई प्रमाणीकरण आवश्यक नहीं।

डाउनलोड प्रक्रिया

GpuLLM स्वचालित रूप से लाइसेंस सहमति, GGUF फ़ाइल स्थान, HTTP रिज़्यूम डाउनलोड और SHA-256 सत्यापन संभालता है।

टोकन आवश्यक? यदि डाउनलोड विफल होता है, तो huggingface.co/settings/tokens पर एक Read टोकन बनाएं।

अनुमान और बेंचमार्किंग

बैकएंड चयन

GpuLLM स्वचालित रूप से सबसे तेज़ बैकएंड का चयन करता है: CUDA 12 → Vulkan → CPU।

बेंचमार्क मेट्रिक्स

  • TTFT — पहला टोकन आने का समय। Tokens/s — जनरेशन थ्रूपुट। विलंबता — कुल प्रतिक्रिया समय।

टिप्स और समस्या निवारण

कीबोर्ड शॉर्टकट

  • Enter या Ctrl+Enter — प्रॉम्प्ट भेजें।

समस्या निवारण

  • 401/403 त्रुटि: सुरक्षित मॉडल। HuggingFace Read टोकन बनाएं।
  • अखंडता जांच विफल: फ़ाइल दूषित। फिर से डाउनलोड करें।
  • HTTP 429: उच्च दर सीमा के लिए टोकन कॉन्फ़िगर करें।
  • धीमा अनुमान: CUDA सक्रिय है या नहीं जांचें। NVIDIA ड्राइवर अपडेट करें।
  • CUDA नहीं मिला: NVIDIA ड्राइवर v545+ इंस्टॉल करें।
  • "Out of Memory": छोटा मॉडल आज़माएं। My Hardware का उपयोग करें।
  • बकवास टेक्स्ट: SHA-256 सत्यापित करें। GGUF प्रारूप सुनिश्चित करें।

शब्दावली

GGUFllama.cpp प्रोजेक्ट द्वारा डिज़ाइन किया गया क्वांटाइज़्ड LLM स्टोर करने का फ़ाइल प्रारूप। GpuLLM विशेष रूप से GGUF का उपयोग करता है।
क्वांटाइज़ेशनसंख्यात्मक सटीकता कम करके मॉडल की मेमोरी खपत कम करने की तकनीक। सामान्य प्रारूप: Q8_0, Q4_K_M, Q2_K।
TTFTTime-To-First-Token — प्रॉम्प्ट भेजने और पहला शब्द प्राप्त करने के बीच का विलंब।
MMLU57 विषयों को कवर करने वाला मानकीकृत बेंचमार्क। GpuLLM में 100 प्रश्न शामिल हैं।
C-Evalचीनी शैक्षणिक मूल्यांकन बेंचमार्क।
SHA-256डाउनलोड की गई मॉडल फ़ाइलों की अखंडता सत्यापित करने के लिए क्रिप्टोग्राफ़िक हैश फ़ंक्शन।
DPAPIWindows Data Protection API — संवेदनशील डेटा को एन्क्रिप्ट करता है।
VRAMVideo RAM — GPU पर समर्पित मेमोरी। बड़े मॉडलों को अधिक VRAM की आवश्यकता होती है।
बैकएंडमॉडल अनुमान चलाने वाला कम्प्यूटेशन इंजन। CUDA, Vulkan, CPU का समर्थन करता है।
