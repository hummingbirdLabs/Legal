GpuLLM 能做什么

GpuLLM 是一款 Windows 桌面应用，将你的 PC 变成完整的本地 AI 工作站。所有计算都在你的硬件上运行——数据永不离开你的设备。

🧠

本地运行大模型

加载 GGUF 格式模型并直接在 GPU 上与之对话。支持 CUDA、Vulkan 和 CPU 后端，具备自动回退功能。

📊

监控硬件

实时 GPU 利用率、显存、温度、功耗、CPU 使用率和系统内存——全部集成在一个仪表板中，配备 60 秒迷你趋势图。

🔬

评测模型

测量推理速度（TTFT、tokens/s、延迟）并使用内置的 MMLU 和 C-Eval 题库评估模型准确率。

📦

下载与管理模型

浏览来自 HuggingFace 的 13 个精选模型目录，支持断点续传和 SHA-256 验证下载，或导入你自己的 GGUF 文件。

💬

双模型对话

在单次对话中并排对比两个模型。LLM1 回答，LLM2 评审——双方都能看到完整的上下文。

😺

棱镜猫吉祥物

一只实时动画 GPU 精灵，根据 GPU 工作负载改变颜色和情绪。

100% 本地与隐私保护：模型下载后，推理无需互联网连接。无遥测、无云端依赖、无数据上传。

系统要求

安装前请确认你的电脑满足以下要求。

操作系统Windows 10（21H2 或更高版本）/ Windows 11
GPUNVIDIA GTX 10 系列或更新（CUDA 12），或任何兼容 Vulkan 的 GPU
显存小型模型（1-3B）最低 4 GB；推荐 8 GB；7B+ 模型需 16+ GB
系统内存最低 8 GB；大于 7B 的模型推荐 16 GB
存储每个模型文件 1-20 GB；推荐使用 SSD 以加快模型加载速度
网络仅下载模型时需要；所有推理均离线运行
没有强大的 GPU？GpuLLM 也支持纯 CPU 系统——只是推理速度较慢。从 1-3B 参数的模型开始，如 Llama 3.2 1B 或 DeepSeek-R1-Distill 1.5B。

快速入门——首次设置

按照以下步骤，从零开始运行你的首次本地大模型推理。

  1. 下载并安装 GpuLLM

    从 Microsoft Store 获取 GpuLLM。应用为自包含包——无需额外运行时或 CUDA 工具包。

  2. 配置 HuggingFace 访问令牌（可选）

    打开设置并输入 HuggingFace Read 令牌。这仅适用于受限模型（如 Qwen、Meta-Llama）。来自 bartowski/* 的社区模型无需令牌即可使用。

    如何创建令牌：前往 huggingface.co/settings/tokens，点击New token，选择类型Read，命名（如"GpuLLM"），复制生成的令牌。将其粘贴到 GpuLLM 设置中。令牌通过 Windows DPAPI 加密存储，绝不会记录到磁盘。
  3. 下载模型

    导航到模型库，浏览精选目录，在任意模型卡片上点击下载。使用我的硬件按钮筛选与 GPU 显存兼容的模型。

  4. 加载模型

    下载完成后，在模型卡片上点击加载。模型权重被加载到 GPU 内存中并常驻，以实现快速推理。

  5. 开始对话

    切换到聊天评测，输入提示词，按Enter键。模型将实时流式输出回复，同时侧边栏实时更新 GPU 指标。

专业提示：先用较小模型（如 Llama 3.2 1B 或 DeepSeek-R1-Distill 1.5B）验证一切正常运行，再使用更大的模型。

常见使用场景

不确定从何开始？以下是一些典型工作流程供参考。

🎯 "我想找到最适合我 GPU 的最快模型"

前往模型库，点击我的硬件筛选兼容模型，下载几个，逐一加载，在聊天评测中对比它们的 tokens/s 得分。每次推理后查看评测 KPI 卡片。

🤝 "我想并排对比两个模型"

加载两个模型，切换到聊天评测，切换双模型模式，将一个设为回答者，另一个设为评审者。两个模型响应相同的提示词——实时对比答案。

📊 "我想了解模型有多准确"

加载模型，导航到评估，选择 MMLU 或 C-Eval，点击开始。你将获得各学科准确率得分和难度级别分布——可导出为 Markdown。

💰 "我想看看相比云端 API 节省了多少钱"

在聊天评测中运行任意推理。内置成本估算器自动将本地推理成本（电费+硬件折旧）与 OpenAI 和 Anthropic API 定价进行对比。查看会话报告了解累计节省额。

📥 "我已有 GGUF 文件——可以直接使用吗？"

可以。在模型库中点击导入模型并选择你的 GGUF 文件，或直接将其拖放到窗口上。GpuLLM 将其添加到库中，无需重新下载。

应用导航

GpuLLM 有五个主要页面，可通过左侧边栏访问。点击任意图标切换页面。

聊天评测

主要的推理和评测页面。在这里你可以：

  • 与模型对话：在任务输入框中输入内容，按Enter（或Ctrl+Enter）发送。模型逐 Token 流式输出回复。
  • 监控 GPU 指标：右侧面板显示实时 GPU 利用率、显存使用、温度、功耗、CPU 使用率和系统内存——全部带有 60 秒迷你趋势图。
  • 调整推理参数：使用顶部的滑块为每次请求设置最大 Token 数和温度。
  • 查看评测结果：每次推理后，KPI 卡片显示 TTFT（首 Token 延迟）、tokens/秒、延迟、后端链路和 GPU 峰值指标。
  • 导出对话：点击导出 PDF将对话和评测报告保存为 A4 PDF 文档。
  • 复制消息：每条消息气泡都有复制按钮和时间戳（HH:mm:ss）。

双模型对话

在聊天评测中切换双模型模式以启用双模型接力推理：

  • 从下拉菜单选择主模型（LLM1）和辅助模型（LLM2）。
  • LLM1 接收你的提示并生成答案。
  • LLM2 自动接收完整对话上下文并提供评审或第二意见。
  • 消息按角色标记：LLM1 · 回答者和LLM2 · 评审者。

评估

使用标准化题库对已加载的模型运行结构化评测：

  • MMLU：涵盖 57 个学科（理工、人文、社科）的 100 道抽样题目。
  • C-Eval：来自中文学术评估基准的 100 道抽样题目。
  • 页面分三阶段：配置（选择模型+基准）→ 运行中（实时进度）→ 结果（按类别和难度显示准确率）。
  • 评分前先执行预热推理，确保模型完全加载。

模型库

浏览、下载和管理你的 GGUF 模型集合：

  • 筛选标签：按类别筛选——编程、聊天或推理。
  • 我的硬件：一键筛选，仅显示 GPU 可运行的模型。兼容模型显示绿色勾号。
  • 下载：点击卡片的下载按钮。GpuLLM 自动处理许可同意、文件列表、断点续传下载和 SHA-256 验证。
  • 导入：拖放或点击"导入模型"添加任意已有的 GGUF 文件。
  • 管理：打开文件夹、复制路径或从卡片加载/卸载模型。

设置

按偏好配置应用：

  • 主题：选择你喜欢的视觉主题。
  • 下载目录：选择模型文件的存储位置。
  • HuggingFace 令牌：输入或清除 HuggingFace 访问令牌（安全存储在 Windows 凭据管理器中）。
  • 法律链接：访问隐私政策和第三方声明。

了解模型下载

受限模型 vs. 开放模型

模型卡片显示徽章以指示是否需要认证：

  • 🔒 受限——需要 HuggingFace 访问令牌。常见于 Qwen、meta-llama 等官方仓库。
  • 开放（无徽章）——无需认证。来自 bartowski/*、lmstudio-community/*、TheBloke/* 的社区重新打包版本。

下载流程

当你点击下载时，GpuLLM 自动执行：

  1. 检查许可同意（宽松许可证静默通过；受限许可证显示对话框）。
  2. 查询 HuggingFace API 定位正确的 GGUF 文件。
  3. 支持 HTTP 断点续传下载（.part 临时文件）。
  4. 根据 HuggingFace LFS 元数据验证 SHA-256 哈希值。
需要令牌？如果下载失败并提示"需要访问令牌"，请在 huggingface.co/settings/tokens 创建 HuggingFace Read 令牌并输入到设置中。令牌存储在 Windows 凭据管理器（DPAPI 加密）中，绝不会记录到磁盘。

管理下载

  • 取消：在下载中的卡片上点击取消。部分 .part 文件保留以便后续续传。
  • 更改位置：前往设置 → 下载目录选择新文件夹。
  • 打开文件夹：在已下载的卡片上点击文件夹图标，在资源管理器中打开文件位置。
  • 导入 GGUF：使用"导入模型"按钮或拖放添加已有的 GGUF 文件。

推理与评测详解

后端选择

GpuLLM 自动检测硬件并按以下顺序选择最快可用后端：CUDA 12 → Vulkan → CPU。无需手动配置——每次推理后评测结果中显示后端链路。

模型加载策略

在模型卡片上点击加载后，模型权重被预加载到 GPU/CPU 内存并常驻。所有后续推理调用复用预加载的权重，完全跳过磁盘 I/O，实现快速响应。同一时间只能加载一个模型（双模型模式下为两个）。

评测指标

每次推理后，GpuLLM 报告：

  • TTFT——首 Token 延迟：第一个回复 Token 出现的等待时间。
  • Tokens/s——首个 Token 之后的生成吞吐量。
  • 延迟——端到端总响应时间。
  • GPU 峰值——推理期间的最大 GPU 利用率、显存使用和温度。
  • 后端链路——使用了哪个后端（例如 "CUDA 12 → Vulkan → CPU 回退"）。

会话报告汇总自应用启动以来的所有推理指标（累计 Token 数、平均延迟、GPU 峰值值）。

导出选项

  • 导出 PDF：将完整对话和会话评测报告保存为 A4 PDF。
  • 复制消息：每条消息都有复制按钮，可导出到剪贴板。

技巧与故障排除

键盘快捷键

  • Enter 或 Ctrl+Enter——在聊天评测中发送提示词。

最佳实践

  • 从小开始：先用 1-3B 参数模型测试，确认设置正常后再加载需要更多显存的大型模型。
  • 检查显存：使用模型库中的我的硬件筛选器，仅查看适合 GPU 显存的模型。绿色勾号确认兼容性。
  • 温度调优：事实性/编程任务使用较低温度（0.1–0.5）；创意写作使用较高温度（0.7–1.0）。在聊天评测滑块中按任务调整。
  • 最大 Token 数：合理设置——过低会截断回复；过高浪费时间。大多数任务 512–2048 个 Token 即可。
  • 双模型对比：使用双模型模式搭配两个不同模型（如编程模型+通用模型），在同一提示词下对比答案。
  • 评估准确性：使用较低温度（0.1–0.3）运行评估，以获得更具确定性和公平的评分。
  • 令牌安全：HF 令牌通过 Windows DPAPI 加密存储在凭据管理器中——绝不会记录到文件。随时可从 huggingface.co/settings/tokens 撤销。

故障排除

  • 模型下载失败 401/403：模型受限。在 huggingface.co/settings/tokens 创建 HuggingFace Read 令牌并输入到设置中，然后重试。
  • 下载开始但失败并提示"完整性校验失败"：文件传输中损坏。再次点击下载——GpuLLM 启动全新下载。
  • 速率限制 / HTTP 429：HuggingFace 限制匿名 API 访问。配置令牌以获得更高速率限制。
  • 重启后模型未显示为"已下载"：下载目录可能已更改。检查设置 → 下载目录或使用导入按钮。
  • 推理速度慢：确认 CUDA 后端处于激活状态（显示在评测结果中）。如果回退到 CPU，确保 NVIDIA 驱动程序是最新的。
  • 未检测到 CUDA / 后端回退到 CPU：安装或更新 NVIDIA 驱动程序至 545 或更高版本。对于笔记本 GPU，确保 GpuLLM 运行在独立 GPU 上。
  • "内存不足"或模型加载失败：GPU 显存不足。尝试更小模型或更高压缩率的量化版本。使用我的硬件筛选兼容模型。关闭其他占用 GPU 的应用程序。
  • 模型加载但生成乱码：GGUF 文件可能损坏或格式错误。验证下载后的 SHA-256 哈希值。确保模型为 GGUF 格式。
  • 双模型模式显示相同模型两次：需要加载两个不同模型再启用双模型模式。先加载 LLM1，再从模型库加载 LLM2。

术语表

GGUF由 llama.cpp 项目设计的量化大语言模型文件格式。GpuLLM 仅使用 GGUF 格式。
量化通过降低数值精度（如从 16 位降至 4 位）减少模型内存占用的技术。更低的量化使用更少显存，但质量可能略降。常见格式：Q8_0、Q4_K_M、Q2_K。
TTFT首 Token 延迟——发送提示词到收到第一个回复词之间的延迟。越低越好。
Tokens/s首 Token 之后每秒生成的 Token 数。这是衡量推理吞吐量的主要指标。
MMLU大规模多任务语言理解——覆盖 57 个学科的标准化评测基准。GpuLLM 内置 100 道英文题目。
C-Eval中文学术评估基准。GpuLLM 包含 100 道中文题目，用于评估中文模型能力。
HuggingFace LFS大文件存储——基于 Git 的系统，用于在 HuggingFace Hub 存储模型文件。每个文件配备 SHA-256 哈希用于完整性验证。
SHA-256密码学哈希函数，用于验证下载的模型文件完整且未被篡改。
DPAPIWindows 数据保护 API——加密敏感数据（如 HuggingFace 令牌），使其只能由同一 Windows 用户账户解密。
VRAM显存——GPU 专用内存。模型加载和推理均消耗显存。更大的模型需要更多显存。
后端运行模型推理的计算引擎。GpuLLM 支持 CUDA（NVIDIA GPU）、Vulkan（跨厂商 GPU）和 CPU（回退）。
