What GpuLLM Can Do

GpuLLM is a Windows desktop application that turns your PC into a complete local AI workstation. All computation runs on your hardware — nothing leaves your device.

🧠

Run LLMs Locally

Load GGUF-format models and chat with them directly on your GPU. Supports CUDA, Vulkan, and CPU backends with automatic fallback.

📊

Monitor Hardware

Real-time GPU utilization, VRAM, temperature, power draw, CPU usage, and memory — all in one dashboard with 60-second sparkline charts.

🔬

Benchmark Models

Measure inference speed (TTFT, tokens/sec, latency) and evaluate model accuracy with built-in MMLU and C-Eval question sets.

📦

Download & Manage Models

Browse curated models from HuggingFace, download with resume support and SHA-256 verification, and import your own GGUF files.

💬

Dual-Model Chat

Compare two models side-by-side in a single conversation. LLM1 answers, LLM2 reviews — both see the full context.

😺

Prism Cat Mascot

A live animated GPU spirit that changes color and emotion based on your GPU workload.

100% Local & Private: Once models are downloaded, no internet connection is required for inference. No telemetry, no cloud dependency, no data upload.

Quick Start — First-Time Setup

Follow these steps to go from zero to running your first local LLM inference.

  1. Download & install GpuLLM

    Get GpuLLM from the Microsoft Store (coming soon). The app is self-contained — no additional runtimes or CUDA toolkit required.

  2. Configure your HuggingFace Access Token (optional)

    Open Settings and enter a HuggingFace Read token. This is only needed for gated models (e.g., Qwen, Meta-Llama, Microsoft Phi). Community models from bartowski/* work without a token.

    See the Setup Guide for step-by-step token creation instructions.

  3. Download a model

    Navigate to Model Library, browse the curated collection, and click Download on any model card. Use the My Hardware button to filter models compatible with your GPU's VRAM.

  4. Load the model

    Once downloaded, click Load on the model card. The model weights are loaded into GPU memory and stay resident for fast inference.

  5. Start chatting

    Switch to Chat Benchmark, type your prompt, and press Enter. The model streams its response in real time while GPU metrics update live in the sidebar.

Pro tip: Start with a smaller model (e.g., Qwen3-1.7B) to verify everything works before moving to larger models.

App Navigation

GpuLLM has five main pages accessible from the left sidebar. Click any icon to switch pages.

Chat Benchmark

The main inference and benchmarking page. Here you can:

  • Chat with a model: Type in the Task Input box and press Enter (or Ctrl+Enter) to send. The model streams its response token-by-token.
  • Monitor GPU metrics: The right panel shows real-time GPU utilization, VRAM usage, temperature, power draw, CPU usage, and system memory — all with 60-second sparkline trends.
  • Adjust inference parameters: Use the sliders at the top to set Max Tokens and Temperature for each request.
  • View benchmark results: After each inference, KPI cards show TTFT (Time-To-First-Token), tokens per second, latency, backend chain, and GPU peak metrics.
  • Export chat: Click Export PDF to save the conversation and benchmark report as an A4 PDF document.
  • Copy messages: Each message bubble has a copy button and timestamp (HH:mm:ss).

Dual-Model Conversation

Toggle Dual Mode in Chat Benchmark to enable two-model relay inference:

  • Select a primary model (LLM1) and a secondary model (LLM2) from the dropdowns.
  • LLM1 receives your prompt and generates an answer.
  • LLM2 automatically receives the full conversation context and provides a review or second opinion.
  • Messages are labeled by role: LLM1 · Answerer and LLM2 · Reviewer.

Evaluation

Run structured benchmarks against loaded models using standardized question sets:

  • MMLU: 100 sampled questions across 57 subjects (STEM, humanities, social sciences).
  • C-Eval: 100 sampled questions from Chinese academic evaluation benchmarks.
  • The page has three phases: Configure (choose model + benchmark) → Running (real-time progress) → Results (accuracy by category and difficulty).
  • A warm-up inference runs first to ensure the model is fully loaded before scoring begins.

Model Library

Browse, download, and manage your GGUF model collection:

  • Filter pills: Narrow by category (Coding, NLP, Vision, Audio, Science) and VRAM requirement.
  • My Hardware: One-click filter to show only models your GPU can run. Compatible models show a green checkmark.
  • Download: Click a card's Download button. GpuLLM handles license consent, file listing, download with resume, and SHA-256 verification automatically.
  • Import: Drag-and-drop or click "Import Model" to add any GGUF file you already have.
  • Manage: Open folder, copy path, or load/unload models from their cards.

Settings

Configure the application to your preferences:

  • Language: Switch between 7 UI languages instantly (English, Japanese, German, French, Spanish, Portuguese, Russian).
  • Theme: Toggle between Light and Dark mode.
  • Download Directory: Choose where model files are stored.
  • HuggingFace Token: Enter or clear your HuggingFace Access Token (stored securely in Windows Credential Manager).
  • Legal links: Access the Privacy Policy and Third-Party Notices.

Understanding Model Downloads

Gated vs. Open Models

Model cards show a badge indicating whether authentication is required:

  • 🔒 Gated (orange lock icon) — Requires a HuggingFace Access Token. Common for official repos from Qwen, microsoft, meta-llama.
  • Open (no badge) — No authentication needed. Community repacks from bartowski/*, MaziyarPanahi/*, etc.

Download Process

When you click Download, GpuLLM automatically:

  1. Checks license consent (silent for permissive licenses; shows dialog for restricted licenses).
  2. Queries HuggingFace API to locate the correct GGUF file.
  3. Downloads with HTTP resume support (.part temporary file).
  4. Verifies SHA-256 hash against HuggingFace LFS metadata.
Token required? If a download fails with "Access Token Required," see the Setup Guide for token creation steps. Your token is stored in Windows Credential Manager (DPAPI-encrypted) and never logged or written to disk.

Managing Downloads

  • Cancel: Click cancel on a downloading card. The partial .part file is kept for later resume.
  • Change location: Go to Settings → Download Directory to pick a new folder.
  • Open folder: On a downloaded card, click the folder icon to open Explorer at the file location.
  • Import GGUF: Use "Import Model" button or drag-and-drop to add existing GGUF files.

Inference & Benchmarking Explained

Backend Selection

GpuLLM automatically detects your hardware and selects the fastest available backend in this order: CUDA 12 → Vulkan → CPU. No manual configuration needed — the backend chain is shown in the benchmark results after each inference.

Model Loading Strategy

When you click Load on a model card, the model weights are pre-loaded into GPU/CPU memory and stay resident. All subsequent inference calls reuse the pre-loaded weights, skipping disk I/O entirely for fast response times. Only one model can be loaded at a time (two in Dual Mode).

Benchmark Metrics

After each inference, GpuLLM reports:

  • TTFT — Time-To-First-Token: How long before the first response token appears.
  • Tokens/s — Generation throughput after the first token.
  • Latency — Total end-to-end response time.
  • GPU Peak — Maximum GPU utilization, VRAM usage, and temperature during inference.
  • Backend Chain — Which backend was used (e.g., "CUDA 12 → Vulkan → CPU fallback").

The session report aggregates metrics across all inferences since app launch (cumulative tokens, average latency, peak GPU values).

Export Options

  • Export PDF: Saves the full chat conversation plus the session benchmark report as an A4 PDF.
  • Copy messages: Each message has a copy button for clipboard export.

Tips & Shortcuts

Keyboard Shortcuts

  • Enter or Ctrl+Enter — Send prompt in Chat Benchmark.

Best Practices

  • Start small: Test with a 1–3B parameter model to verify your setup before loading larger models that require more VRAM.
  • Check VRAM: Use the My Hardware filter in Model Library to see only models that fit your GPU's VRAM. The green checkmark confirms compatibility.
  • Temperature tuning: Lower temperature (0.1–0.5) for factual/coding tasks; higher (0.7–1.0) for creative writing. Adjust per-task in the Chat Benchmark sliders.
  • Max Tokens: Set appropriately — too low cuts off responses; too high wastes time. Most tasks work well with 512–2048 tokens.
  • Dual Model comparison: Use Dual Mode with two different models (e.g., a coding model + a general model) to compare answers on the same prompt.
  • Evaluation accuracy: Run evaluations with lower temperature (0.1–0.3) for more deterministic and fair scoring.
  • Token security: Your HF token is encrypted via Windows DPAPI and stored in Credential Manager — it's never logged to files. Revoke it anytime from huggingface.co/settings/tokens.

Troubleshooting

  • Model download fails with 401/403: The model is gated. Configure a HuggingFace token in Settings, then retry. See the Setup Guide.
  • Download starts but fails with "Integrity check failed": The file was corrupted during transfer. Click Download again — GpuLLM starts fresh.
  • Rate limit / HTTP 429: HuggingFace limits anonymous API access. Configure a token for higher rate limits.
  • Model doesn't show as "Downloaded" after restart: Your download directory may have changed. Check Settings → Download Directory or use the Import button.
  • Inference is slow: Verify the CUDA backend is active (shown in benchmark results). If falling back to CPU, ensure your NVIDIA driver is up to date.
